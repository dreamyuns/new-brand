정렬 버튼
  • 기본값: "최신 예약순" (가장 최근에 예약한 건이 위로).

  • 두 정렬 모두 상태 그룹 고정 적용 → 투숙중 → 투숙예정 → 투숙완료 → 예약취소 순.
  • 체크인 임박순: 투숙중·투숙예정 그룹 내에서 체크인 날짜가 빠른 것이 위. 투숙완료·취소는 날짜 최신순.
정렬 그룹 우선순위 고정. 그룹 내 세부 정렬만 변경됨.

전체 건수

← Transaction Report 전체 건수
9
여행 상태 배지 (카드 좌측 상단)
  • 자사 서버에서 오늘 날짜와 체크인·체크아웃 날짜를 비교해 계산한 값. KAYAK이 직접 제공하는 값이 아님.
  • 투숙중 (청록): 오늘이 체크인 날짜 이상, 체크아웃 날짜 미만인 경우. ← 정렬 1순위
  • 투숙예정 (파란색): 오늘이 체크인 날짜보다 이전인 경우. ← 정렬 2순위
  • 투숙완료 (회청): 오늘이 체크아웃 날짜 이후인 경우 (체크아웃 당일 +1일부터). ← 정렬 3순위
  • 예약취소 (회색): KAYAK 상태가 Cancelled인 경우. ← 정렬 4순위
← 자사 서버 계산: clickStartDate / clickEndDate vs 오늘 날짜. ⚠️ 개발팀 확인 필요 — 서버 or 프론트 계산 여부.
정렬 우선순위: 투숙중(1) → 투숙예정(2) → 투숙완료(3) → 예약취소(4).
10
호텔명
  • KAYAK Transaction Report에서 가져온 호텔 이름. 너무 길면 말줄임(...) 처리.
  • 예약취소 건은 글자색을 회색으로 표시.
← hotelName (Transaction Report Hotel vertical 필드)
11
체크인 ~ 체크아웃 날짜
  • 표시 형식: MM.DD ~ MM.DD. 예: 07.01 ~ 07.03.
  • 예약취소 건은 날짜도 회색 처리.
← bookingStartDate / bookingEndDate (Transaction Report)
12
적립 상태 배지 (카드 우하단 — 금액 왼쪽)
  • KAYAK Transaction Report 상태값을 아래처럼 바꿔 표시한다.
  • 적립예정 (주황): Active 상태 → 아직 체크아웃 전이거나 정산 대기 중.
  • 적립완료 (초록): Approved → KAYAK 정산 완료, 금액 확정.
  • 회수 (회색): Cancelled → 취소로 인해 캐시백이 적립되지 않음. 금액에 취소선으로 표시.
  • ⚠️ 실 데이터 확인(2026-07-06): Cancelled 건에서 캐시백 금액이 0원으로 내려오는 경우 있음. 금액이 0원이어도 취소선 표시를 먼저 적용 — 예약이 취소됐는지를 먼저 확인하고, 취소됐다면 금액과 관계없이 취소선 처리.
  • cashbackType=NONE 공급사 예약 건은 캐시백 내역 자체 미표시 (별도 정책 — Cancelled 건과 다름).
← Transaction Report 상태: Active=적립예정 / Approved=적립완료 / Cancelled=회수. 실 데이터 확인(2026-07-06): Cancelled 건에도 PaymentMonth 값이 있을 수 있으므로 TransactionStatus를 반드시 먼저 체크.
정책서 9-4절. 여행 상태 배지와 분리 표시 (여행 상태: 좌상단 / 적립 상태: 우하단).
13
호텔 이미지
  • 카드 좌측 고정 이미지 영역 (72×전체 높이). KAYAK Static Data Feed에서 khotel ID로 조회한 호텔 대표 사진.
  • 이미지 로드 실패 또는 khotel ID 매핑이 없을 때: 회색 플레이스홀더 + 호텔 아이콘 표시.
← khotel ID → KAYAK Static Data Feed 이미지 URL 조회. ⚠️ 개발팀 확인 필요 — Static Data Feed 이미지 필드 제공 여부 및 연동 방법.
이미지 없을 때 플레이스홀더로 대체. 레이아웃 깨짐 없어야 함.
14
캐시백 금액 (카드 우하단)
  • KAYAK Transaction Report에서 가져온 금액. 자사에서 다시 계산하지 않는다.
  • 상태별 색상: 적립예정(주황) / 적립완료(초록) / 회수(회색 취소선 또는 "회수" 텍스트).
  • 예약취소 건은 금액 자리에 숫자 대신 "회수" 텍스트만 표시.

정책서 9-4절·10-2절. 통화 단위는 선택 통화 기준 (localisedCurrencyCode). Cancelled + 금액 0 케이스는 cashbackType=NONE 미표시 정책과 별개.

화면 주석 — MY02-D1-B1 (예약상세 모달)

진입
모달 진입 조건 및 닫기
  • B 화면에서 여행 카드(trip-card) 어디든 탭 → 해당 예약의 상세 모달이 화면 중앙에 나타난다.
  • 배경 어두운 영역(딤)을 탭하거나 [✕] 버튼을 탭 → 모달이 닫히고 B 화면 목록으로 돌아온다.
  • 데이터 출처: KAYAK Reporting API v2 — Hotel vertical (/api/transactions/hotels)
정책서 10-2절. Reporting API Hotel vertical 필드 사용.
M1
모달 헤더 — "예약 상세" + [✕] 닫기
  • 모달 상단에 "예약 상세" 제목이 표시된다.
  • [✕] 버튼을 탭하면 모달이 닫힌다.
M2
예약 상태 배지 (모달 최상단 좌측)
  • MY02는 transactionType = 10 (Booking) 예약 확정 건만 표시한다. Lead 건(TypeCode=1)은 카드에 나타나지 않는다.
  • 예약 확정 배지 (파란색): 예약이 완료된 건임을 나타낸다.
← transactionType = 10 (Booking 건만 표시)
정책서 10-2절.
M3
적립 상태 배지 (모달 최상단 우측)
  • 💸 적립예정: transactionStatus = Active 상태인 경우.
  • 💸 지급완료: paymentMonth 값이 있는 경우 (개발팀 판단 기준 확인 필요).
  • 취소·만료: transactionStatus = Cancelled 상태인 경우.
← transactionStatus + paymentMonth
정책서 9-4절. 지급완료 판단: paymentMonth 값이 있으면 지급완료. 없으면 Active 상태 = 적립예정.
M4
호텔명 + 위치
  • 호텔명: hotelName (Hotel vertical 전용 필드)
  • 위치: hotelCity + hotelCountry — "도시 · 국가" 형식 표시.
← hotelName / hotelCity / hotelCountry
M5
체크인·체크아웃 + 박수
  • 체크인: bookingStartDate, 체크아웃: bookingEndDate. MY02는 Booking 건만 표시하므로 항상 이 두 필드를 사용한다.
  • 박수: 두 날짜 차이를 자사에서 계산하여 표시. 예: 07.01 → 07.03 = 2박.
← bookingStartDate / bookingEndDate (Booking 전용 필드)
M6
OTA 공급사명 ⚠️ 개발팀 확인사항
  • Booking.com·Agoda·Expedia 등 실제 예약이 이루어진 OTA 이름을 표시한다.
  • Reporting API는 이 값을 직접 주지 않는다. siteBrandCode는 "kayak"·"HotelsCombined" 등 플랫폼명이므로 공급사명이 아님.
  • 권장 방식: 아웃링크 클릭 시점(OUT01)에 자사 DB에 providerName을 저장 → Reporting API clickId로 연결해서 불러오는 방식.
← ⚠️ 개발팀 확인 필요: 아웃링크 클릭 시점에 providerName을 자사 DB에 저장하는 방식 검토 요망.
