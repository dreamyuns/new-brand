체리 아이콘 (자물쇠 없음)

지금까지 아낀 금액 ★

  • 탭 시 → 정렬 선택 팝업(바텀시트) 열림: 최신 예약순 / 체크인 임박순.

전체 건수
  • 모든 상태(Waiting·Approved·Cancelled) 예약 건수 합산 표시. 예: "전체 7건".
← Transaction Report 전체 건수
9
여행 상태 배지 (카드 좌측 상단)
  • 자사 서버에서 오늘 날짜와 체크인·체크아웃 날짜를 비교해 계산한 값. KAYAK이 직접 제공하는 값이 아님.
  • 투숙중 (청록): 오늘이 체크인 날짜 이상, 체크아웃 날짜 미만인 경우. ← 정렬 1순위
  • 투숙예정 (파란색): 오늘이 체크인 날짜보다 이전인 경우. ← 정렬 2순위
  • 투숙완료 (회청): 오늘이 체크아웃 날짜 이후인 경우 (체크아웃 당일 +1일부터). ← 정렬 3순위
  • 예약취소 (회색): KAYAK 상태가 Cancelled인 경우. ← 정렬 4순위
← 자사 서버 계산: bookingStartDate / bookingEndDate vs 오늘 날짜. ⚠️ 개발팀 확인 필요 — 서버 or 프론트 계산 여부.
정렬 우선순위: 투숙중(1) → 투숙예정(2) → 투숙완료(3) → 예약취소(4).
10
호텔명
  • KAYAK Transaction Report에서 가져온 호텔 이름. 너무 길면 말줄임(...) 처리.
  • 예약취소 건은 글자색을 회색으로 표시.
← hotelName (Transaction Report Hotel vertical 필드)
11
체크인 ~ 체크아웃 날짜
  • 표시 형식: MM.DD ~ MM.DD. 예: 07.01 ~ 07.03.
  • 예약취소 건은 날짜도 회색 처리.
← bookingStartDate / bookingEndDate (Transaction Report)
12
적립 상태 배지 (카드 우하단 — 금액 왼쪽)
  • KAYAK Transaction Report 상태값을 아래처럼 바꿔 표시한다.
  • 적립예정 (주황): Active 상태 → 아직 체크아웃 전이거나 정산 대기 중.
  • 적립완료 (초록): Approved → KAYAK 정산 완료, 금액 확정.
  • 회수 (회색): Cancelled → 취소로 인해 캐시백이 적립되지 않음. 금액 행 미표시 — 금액 자리 없이 배지만 표시.
  • ⚠️ 실 데이터 확인(2026-07-06): Cancelled 건에서 캐시백 금액이 0원으로 내려오는 경우 있음. Cancelled 건은 금액과 무관하게 금액 행 자체 숨김 — TransactionStatus=Cancelled이면 금액 없이 "회수" 배지만 표시.
  • cashbackType=NONE 공급사 예약 건은 캐시백 내역 자체 미표시 (별도 정책 — Cancelled 건과 다름).
← Transaction Report 상태: Active=적립예정 / Approved=적립완료 / Cancelled=회수(금액 행 미표시). TransactionStatus를 먼저 체크 — Cancelled면 cashbackAmountLocalised 값 무관하게 금액 숨김.
정책서 9-4절. 여행 상태 배지와 분리 표시 (여행 상태: 좌상단 / 적립 상태: 우하단).
13
호텔 이미지
  • 카드 좌측 고정 이미지 영역 (72×전체 높이). KAYAK Static Data Feed에서 khotel ID로 조회한 호텔 대표 사진.
  • 이미지 로드 실패 또는 khotel ID 매핑이 없을 때: 회색 플레이스홀더 + 호텔 아이콘 표시.
← khotel ID → KAYAK Static Data Feed 이미지 URL 조회. ⚠️ 개발팀 확인 필요 — Static Data Feed 이미지 필드 제공 여부 및 연동 방법.
이미지 없을 때 플레이스홀더로 대체. 레이아웃 깨짐 없어야 함.
14
캐시백 금액 (카드 우하단)
  • KAYAK Transaction Report에서 가져온 금액. 자사에서 다시 계산하지 않는다.
  • 상태별 금액 표시: 적립예정(주황 금액 표시) / 적립완료(초록 금액 표시) / 회수(금액 행 미표시).
  • 예약취소(Cancelled) 건은 캐시백 금액 행 자체를 표시하지 않는다. 카드 우하단에 "회수" 배지(eb-refund)만 표시.
← cashbackAmountLocalised (Transaction Report). 미표시 조건: ① cashbackType=NONE ② Cancelled 건 (금액 행 전체 숨김, TransactionStatus 기준 우선 처리). cashbackAmountLocalised=null/0도 숨김.
정책서 9-4절·10-2절. 통화 단위는 선택 통화 기준 (localisedCurrencyCode). Cancelled + 금액 0 케이스는 cashbackType=NONE 미표시 정책과 별개.
15
[여행 N개 더보기 ∨] 버튼
  • 처음 화면에는 최대 4건만 보인다. 버튼을 탭 하면 4건씩 추가로 불러온다.
  • 버튼 문구: "여행 N개 더보기 ∨" → N = 남은 건수. 예: "여행 3개 더보기".
  • 모든 건을 다 불러오면 버튼이 사라진다.
기본 4건 표시 → 탭 시 4건씩 추가 로드. 전체 로드 완료 시 버튼 미노출.

화면 주석 — MY02-D1-B1 (예약상세 모달)

진입
모달 진입 조건 및 닫기
  • B 화면에서 여행 카드(trip-card) 어디든 탭 → 해당 예약의 상세 모달이 화면 중앙에 나타난다.
  • 배경 어두운 영역(딤)을 탭하거나 썸네일 우측 상단 [✕] 버튼을 탭 → 모달이 닫히고 B 화면 목록으로 돌아온다.
  • 데이터 출처: KAYAK Reporting API v2 — Hotel vertical (/api/transactions/hotels)
정책서 10-2절. Reporting API Hotel vertical 필드 사용.
M1
썸네일 이미지 + [✕] 닫기 버튼
  • 모달 상단에 호텔 썸네일 이미지가 풀 너비로 표시된다 (목록 카드와 동일한 이미지 소스).
  • [✕] 닫기 버튼이 이미지 오른쪽 위에 반투명 원형 버튼으로 겹쳐서 나타난다.
  • 탭하면 모달이 닫히고 B 목록 화면으로 돌아온다.
확정 디자인 기준. 썸네일 위 오버레이 방식.
M2
예약 상태 배지 — 썸네일 이미지 위 좌측
  • 썸네일 이미지 왼쪽 아래에 예약 상태 배지가 겹쳐서 나타난다.
  • 예약취소 (회색, dtb-cancel): transactionStatus = Cancelled인 경우 — 투숙 날짜 계산 없이 이 배지 하나만 표시. M3 적립상태 배지 미표시.
  • 비취소 건 (dtb-booking): 투숙예정 / 투숙중 / 투숙완료 — bookingStartDate / bookingEndDate와 오늘 날짜 비교로 자사에서 계산. (B 케이스 CN9 참조)
← transactionStatus = Cancelled 우선 체크. Cancelled이면 dtb-cancel "예약취소" 단독. 비취소이면 bookingStartDate/bookingEndDate 기준 투숙 상태 계산
취소 건은 "예약취소" 배지 하나만 (확정 2026-07-09). 정책서 10-2절.
M3
적립 상태 배지 — 예약 상태 배지 옆 나란히
  • 예약 상태 배지 오른쪽에 나란히 표시된다. B 케이스 목록 카드의 적립 상태 배지(CN12)와 동일한 로직.
  • 적립예정 (노란색): transactionStatus = Active인 경우.
  • 적립완료 (초록색): transactionStatus = Approved — KAYAK 정산 완료.
  • Cancelled 건: 이 배지 미표시. M2에 "예약취소" 배지가 단독으로 표시되므로 M3는 나타나지 않는다.
← transactionStatus. Cancelled 건은 M3 미표시. 비취소 건: Active=적립예정 / Approved=적립완료
정책서 v0.42 9-4절·10-2절. 취소 건 M3 미표시 확정 2026-07-09. B 케이스 CN12 참조.
M4
호텔명 + 위치 — 썸네일 이미지 위 하단
  • 썸네일 이미지 하단에 흰색 텍스트로 겹쳐서 나타난다.
  • 호텔명: hotelName (굵은 흰색 글자)
  • 위치: hotelCity + hotelCountry — "도시, 국가" 형식으로 표시. 예: "서귀포시, 제주특별자치도"
← hotelName / hotelCity / hotelCountry
위치 표시 형식 변경: 기존 "도시 · 국가" → "도시, 국가" (쉼표 구분). 확정 디자인 기준.
M5
체크인 · 체크아웃 날짜 + 박수
  • 체크인: bookingStartDate, 체크아웃: bookingEndDate.
  • 박수: 두 날짜 차이를 자사에서 계산 후 "(N박)" 형식으로 체크아웃 옆에 표시.
  • 표시 형식: "YYYY.MM.DD — YYYY.MM.DD (N박)"
← bookingStartDate / bookingEndDate (Booking 전용 필드)
M6
D-DAY 배지 — 체크인까지 남은 일수
  • 체크인/체크아웃 행 오른쪽에 주황색 원형 배지로 표시된다.
  • 계산: 오늘 날짜 기준, 체크인까지 남은 일수. 예: 5일 남으면 "D-5".
  • D ≥ 1 (체크인 당일 포함 이전)일 때만 배지가 나타난다.
  • 체크인 당일 이후 (D ≤ 0, 즉 투숙 중·완료 상태)에는 배지를 표시하지 않는다.
확정 디자인 신규 항목. 오늘 날짜는 클라이언트 기기 기준으로 계산.

예약처 ⚠️ 개발팀 확인사항

← ⚠️ 개발팀 확인 필요: 아웃링크 클릭 시점에 providerName을 자사 DB에 저장하는 방식 검토 요망.
라벨명 "예약처" (기존 "OTA 공급사"에서 변경, 확정 디자인 기준). Reporting API에 없는 필드 — 별도 저장 로직 필요.

캐시백 금액 (초록 강조)

M11
적립예정일 / 적립완료 — 상태별 분기 표시
  • 적립 상태에 따라 이 항목의 라벨과 값이 달라진다.
  • Active (적립예정) 상태: 라벨 "적립예정일" + 날짜 값 (예: 2026.12.12). bookingEndDate(체크아웃일)를 기준으로 자사에서 계산한 날짜.
  • Approved (지급완료) 상태: 라벨 "적립완료" + 값 없음 (또는 완료 날짜 표시 — 개발 시 확인).
  • Cancelled 상태: 이 행 자체를 화면에 표시하지 않음.
← 상태별 분기: Active → 날짜 계산 표시 / Approved → "적립완료" / Cancelled → 미표시
확정 디자인 기준 (기존 "정산 예정월" 항목에서 변경). 정책서 v0.42 10-2절.
