1
👤 사용자 아이콘 버튼 (좌측 상단)

  • 로그인 전 A 케이스의 [로그인] + [⚙️] 버튼 자리를 대체. 원형 반투명 버튼.

체리 아이콘 (자물쇠 없음)

공통 주의문구 — "나의 여행" 타이틀 하단 고정
  • "나의 여행" 타이틀 바로 아래에 항상 표시. 카드 목록 위에 고정.
  • 문구: "예약 직후 포인트 내역은 최대 4일 이내 반영됩니다"
  • KAYAK Reporting API는 예약 직후 데이터가 불규칙하게 반영되는 특성이 있어 예약 후 최대 4일 이내에 목록에 나타날 수 있음을 안내하는 문구.
포인트_정책서 v0.1 2-5절·5-2절. 항상 고정 표시 (조건 없음).
10
여행 상태 배지 — +4일 이후 카드만 표시
  • 예약일(bookingDate) 기준으로 +4일 이후인 카드에만 여행 상태 배지가 나타난다. +4일 이전 카드에는 없음.
  • 자사 서버에서 오늘 날짜와 체크인·체크아웃 날짜를 비교해서 계산한 값. KAYAK이 직접 제공하는 값이 아님.
  • 투숙중 (청록): 오늘이 체크인 이상, 체크아웃 미만.
  • 투숙예정 (파란색): 오늘이 체크인 이전.
  • 투숙완료 (회색): 오늘이 체크아웃 이후.
  • 예약취소 (연회색): KAYAK 상태가 Cancelled인 경우.
← 자사 서버 계산: bookingStartDate / bookingEndDate vs 오늘. +4일 이전 카드는 이 배지 없음.
포인트_정책서 v0.1 2-5절. +4일 이전 미표시 확정.
11
호텔명
  • KAYAK Transaction Report에서 가져온 호텔 이름. 너무 길면 말줄임(...) 처리.
  • 예약취소 건은 글자색을 회색으로 표시.
← hotelName (Transaction Report Hotel vertical)
12
체크인 ~ 체크아웃 날짜 — +4일 이후 카드만 표시
  • 표시 형식: MM.DD ~ MM.DD. 예: 07.01 ~ 07.03.
  • +4일 이전 카드에는 이 날짜 행이 나타나지 않는다. KAYAK에서 예약 직후에는 날짜 데이터가 불안정하게 내려올 수 있기 때문.
  • 예약취소 건은 날짜도 회색으로 표시.
← bookingStartDate / bookingEndDate. +4일 이전 미표시.

포인트 상태 배지
  • 적립예정 (주황): 체크아웃+7일이 아직 안 지난 상태. +4일 이전·이후 카드 모두 표시.
  • 지급완료 (초록): 체크아웃+7일 배치 처리 완료. +4일 이후 카드에만 표시.

포인트 금액 (카드 우하단)
  • KAYAK Transaction Report에서 가져온 금액. 자사에서 다시 계산하지 않는다. 1KRW = 1P 고정.
  • 표시 형식: N P. 예: 84,000 P.
  • 상태별 색상: 적립예정(주황) / 지급완료(초록) / 취소·만료(금액 행 자체 없음).
← cashbackAmountLocalised × 1P/KRW. 취소·만료 건은 금액 행 미표시.
포인트_정책서 v0.1 2-2절·2-4절.
