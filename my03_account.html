진입
마이페이지 > [계정관리] 탭 → 별도 전체화면
  • 로그인 전용 화면. 비로그인 진입 불가.
  • 기존에 저장된 정보가 있으면 각 필드에 미리 채워진 상태로 진입 (이름·성·국적·전화번호·마케팅 동의).
정책서 10-3절.

← 뒤로가기

이름 필드 (First Name, 영문, 선택)

  • 위반 문자 포함 시 유효성 오류 (MY03-C 참조). 유효 입력 완료 시 우측 ✓ 표시.

정책서 10-3절 (이름: 영문, 선택).
4
성 필드 (Last Name, 영문, 선택)
  • 이름 필드와 동일 유효성 규칙 적용. 영문 알파벳 + 공백만 허용.
  • 이름·성 동시 오류 시 두 필드 모두 에러 표시.
← placeholder: 예) Yoon
정책서 10-3절 (성: 영문, 선택).
5
이메일 필드 (읽기전용) ★
  • 로그인 인증에 사용한 이메일 주소. 수정·변경 불가.
  • 시각 처리: 회색 배경(#f8f8f6) + 커서 not-allowed + 우측 "✓ 인증완료" 초록 배지 고정 표시.
정책서 10-3절 (이메일 읽기전용, 변경 불가).
6
국적 필드 (바텀시트 선택, 선택)
  • 탭 시 국적 선택 바텀시트(MY03-B) 슬라이드업.
  • 선택 완료 시 바텀시트 자동 닫힘 → 필드에 국기 아이콘 + 국가명 표시 (예: 🇰🇷 대한민국).
  • 미선택 상태: 플레이스홀더 "국적을 선택해주세요" 회색 표시.
← placeholder: 국적을 선택해주세요
정책서 10-3절 (국적: 바텀시트 선택).
7
전화번호 필드 (선택)
  • 좌측 국가코드 선택 버튼 + 우측 번호 직접 입력 2분할 구조.
  • 국가코드와 국적 선택은 독립적으로 설정 가능 (연동 없음).
← placeholder: 예) 01012345678
정책서 10-3절 (국가코드 선택 + 번호 직접 입력).
8
국가코드 선택 버튼
  • 탭 시 국가코드 선택 바텀시트 팝업. 기본값: 🇰🇷 +82 (대한민국).
  • 국적 필드 선택값과 별도 독립 설정 — 국적 변경해도 국가코드 자동 변경 없음.
← 기본값: +82 (한국)
정책서 10-3절 (전화번호: 국가코드 선택 + 번호 직접 입력).
9
마케팅 수신 동의 토글 ★
  • ON/OFF 토글 스위치. 이메일·SMS 마케팅 정보 수신 동의 여부 설정.
  • 초기값: 회원가입(AUTH) 시 선택한 마케팅 동의값과 자동 동기화 (가입 시 체크하면 ON, 미체크면 OFF).
  • 토글 변경 후 [저장] 탭 시에만 서버에 반영.
정책서 10-3절 (회원가입 시 선택값과 자동 동기화).

  • 탭 시 마케팅 약관 전문 바텀시트 팝업.
  • 내용 확인 후 [✕] 또는 딤 영역 탭 시 바텀시트 닫힘. 동의 상태 변경 없음.

  • 유효성 오류 시: 오류 필드에 빨간 테두리 + 에러 메시지 표시, 저장 비활성 (MY03-C 참조).

화면 주석 — MY03-B (국적 선택)

진입
국적 필드 탭 → 바텀시트 슬라이드업
  • MY03-A에서 국적 필드 탭 시 슬라이드업. 딤 영역 탭 또는 [✕] 탭 시 선택 없이 닫힘.
  • 국적 선택 즉시 바텀시트 자동 닫힘 → 국적 필드에 선택 국기 + 국가명 반영.
정책서 10-3절 (국적: 바텀시트 선택).
1
국적 선택 바텀시트
  • 전체 국가 목록이 많으므로 검색 필드 포함.
  • 목록 구성: "인기 국적" 섹션 (한국·미국·일본·중국 고정 상단) + "전체" 섹션 (알파벳 순).
정책서 10-3절 (국적: 바텀시트 선택).
2
국가 검색 필드
  • 한국어 국가명 및 영어 국가명 모두 검색 지원 (예: "한국", "Korea" 모두 대한민국 노출).
  • 텍스트 입력 즉시 목록 실시간 필터링.
정책서 10-3절 (국적 바텀시트 — 검색 포함).
3
현재 선택 국적 — ✓ 표시
  • 현재 저장된 국적 항목에 인디고 체크마크(✓)와 텍스트 색상 강조.
  • 다른 항목 탭 시 즉시 선택 → 바텀시트 자동 닫힘 → 국적 필드에 국기+국가명 반영.
인기 국적 섹션: 한국·미국·일본·중국 (상위 4개 고정 노출). 정책서 10-3절.

화면 주석 — MY03-C (유효성 오류)

트리거
[저장] 탭 → 유효성 검사 실행
  • [저장] 버튼 탭 시 모든 입력 필드 일괄 유효성 검사 실행.
  • 오류 필드: 빨간 테두리(#ef4444) + 연분홍 배경(#fef2f2) + 필드 하단 에러 메시지 표시.
  • 오류가 있는 첫 번째 필드로 자동 스크롤 이동.
정책서 10-3절 (저장 전 필드별 유효성 검사).
1
이름 오류 — 특수기호·숫자 입력 시
  • 허용 문자: 영문 알파벳(A-Z, a-z) + 공백. 그 외 전부 불허 (숫자·특수기호·한글 포함).
  • 위반 시 에러 메시지: "영문 알파벳만 입력 가능합니다 (특수기호·숫자 제외)"
  • 시각 처리: 필드 테두리 빨간색 + 배경 연분홍(#fef2f2).
허용: 영문(A-Z, a-z) + 공백. 불허: 숫자·특수기호·한글 전부. 정책서 10-3절.
2
성 오류 — 특수기호·숫자 입력 시
  • 이름 필드와 동일 유효성 규칙 적용.
  • 에러 메시지: "영문 알파벳만 입력 가능합니다 (특수기호·숫자 제외)"
  • 이름·성 동시 오류 시 두 필드 모두 에러 상태 표시.
정책서 10-3절 (이름·성 동시 오류 시 두 필드 모두 에러 표시).
3
[저장] 버튼 — 비활성 상태
  • 유효성 오류가 하나라도 있는 동안 [저장] 버튼 회색 비활성(disabled) 처리 (배경 #e5e7eb, 텍스트 #9ca3af).
  • 오류 필드를 모두 수정 완료 시 [저장] 버튼 인디고 활성 상태로 자동 복귀.
정책서 10-3절 (오류 시 저장 버튼 비활성).
오류 해결 흐름: 잘못된 이름 입력 → 에러 표시 → 영문으로 수정 → 에러 해제 → [저장] 활성화
