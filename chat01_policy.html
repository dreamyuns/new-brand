제1조

개요

1-1 목적

자연어 입력(텍스트·음성)을 통해 호텔 검색 조건을 파싱하고, KAYAK API 결과를 AI 요약과 함께 제공하는 챗봇 검색 기능의 정책을 정의한다.

1-2 MVP 범위
구분포함제외
입력 방식텍스트 입력음성 입력 (추가 논의), 사진 첨부 (추가 논의)
검색 대상호텔 검색 (KAYAK API)항공, 렌터카, 관광명소, 맛집
결과인기순 상위 3개 카드S01 전체 결과는 CTA로 연결
대화세션 내 다중 턴 대화세션 간 히스토리 서버 저장 (추가 논의)

4-1 진입 방법 v0.2 전면 개정

기존 FAB(하단 우측 원형 버튼) 폐기. 대표님 피드백 반영: "AI 검색 버튼이 목적지칸 좌측에 있으면 혼동" → 메인 검색폼에서 분리하여 해소.

▶ 공통 진입점 (C안·D안 동일)
진입 경로설명
GNB 우측 [✦ AI] 배지전 화면 GNB에 공통 노출. 탭 → 채팅창 바로 오픈 (인풋 없음)
▶ 메인 진입 UI — C안 vs D안 (택일 논의 중 ⚠️ 미결 #15)
안구조설명
C안
검색폼 + 하단 AI 배너
[ 메인 검색폼: 목적지·날짜·인원·검색 ] [ ✦ "강남 2인 1박 호텔 찾아줘" → ]
 일반 검색폼 우선 노출. AI는 검색버튼 하단 알약형 배너로 보조 진입.
배너 텍스트: 추천검색어 2.4초 간격 페이드 순환.
배너 탭 → 챗봇 오픈 (배너 문구는 입력창에 들어가지 않음)
D안
AI 검색 | 일반 검색 탭
[ ✦ AI 검색 | 일반 검색 ] ← 기본=AI 검색 ┌──────────────────────────────┐ │ ✦ 가고 싶은 숙소를 AI로 검색 → │ └──────────────────────────────┘ 🔍 강남 2인 1박 20만원대 호텔 🔍 제주 주말 풀빌라 2박 찾아줘
 메인 진입 시 기본 탭 = AI 검색 (AI 전면 부각 쇼케이싱).
자연어 입력창 + 추천질문 칩 3~4개(세로 리스트).
입력 후 [→]/Enter → 챗봇이 열리며 해당 문구로 바로 검색.
추천질문 칩 탭 → 그 질문으로 바로 검색.
일반 검색 탭 탭 시 기존 구조화 검색폼으로 전환.

검색결과(S01) 재검색바 탭 → 재검색 슬라이드다운(S01-A)도 C안/D안 동일 구조 적용.

챗봇 동작 자체(대화·결과·닫기 등)는 C안·D안 완전히 동일. UI 진입 방식만 다름.

5-2 카드 수 및 정렬

표시 수: 3개 고정

정렬 기준: KAYAK API 응답 기본 순서 (KAYAK 인기순) 상위 3개

5-3 카드 클릭 동작
  • 카드 탭 → 채팅창 자동 닫힘 + S01(검색결과 목록) 이동
  • 클릭한 숙소를 S01 목록 최상단에 노출. 나머지 동기화 결과는 그 아래 정렬
  • Phase 1 정책서 4-5절(호텔명 선택 동작) · 5-9절(숙소명 검색 처리 정책)과 동일 로직 적용
5-4 S01 동기화 정책 및 "전체 결과 보기" CTA ★ 개발자 필독

동기화 정책:

  • 챗봇이 KAYAK API를 호출하는 시점에 채팅창(카드 3개)과 S01(전체 목록)에 동시 동기화하여 결과 표시
  • S01은 재호출 없이 이미 준비된 결과를 즉시 표시
  • S01 상단 검색바(도시·날짜·인원)도 챗봇 파싱 조건으로 채워진 상태로 이동
  • 기본값이 적용된 파라미터(날짜 D+7 등)도 S01 검색바에 동일하게 반영

"전체 결과 보기" CTA / ✕ 닫기: v0.2 변경

  • 탭 → 채팅창만 닫힘 (S01로 별도 이동하는 것이 아님)
  • 챗봇 검색 시점에 S01이 이미 동기화돼 있어, 닫으면 배경의 S01이 즉시 보임
  • "전체 결과 보기": 탭한 숙소 없으므로 S01 목록은 KAYAK 인기순 기본 정렬
5-5 챗봇 재진입 시 대화 유지
  • S01 이동 후 GNB 챗봇 버튼 재탭 시 → 이전 채팅 내용이 유지된 상태로 채팅창 재오픈
  • 새 대화가 아닌 기존 대화에서 이어서 후속 질문 가능
  • 채팅창 내 대화 내용은 브라우저 세션이 유지되는 동안 보존
  • 새 대화를 원하면 더보기(···) → "새 대화창" 선택
제6조

추천 검색어 정책 (LLM 자동 생성)

6-1 생성 방식
  • LLM이 결과 도출 후 현재 파싱된 파라미터 기반으로 후속 추천 질문 자동 생성
  • 개수: 4개 고정
  • 생성 방식: 결과 도출 LLM 호출에 추천 검색어 생성을 함께 요청 (단일 호출)
  • 표시 위치: "전체 결과 보기" CTA 하단 → 연결 문장 후 리스트
6-2 생성 가이드라인 (시스템 프롬프트)
[포함] - 현재 조건에서 편의시설 변형 (무료취소, 조식포함, 반려동물 등) - 성급 변경 (4성급 → 5성급 등) - 가격대 조정 (예산 상향/하향) - 날짜 변경 (다음 주, 주말 등) - 숙소 유형 변경 (호텔 → 펜션/리조트 등) [제외 — 범위 밖] - 항공편 관련 질문 - 관광명소, 맛집, 교통편 - KAYAK API에 없는 파라미터 조건

8-2 지역 없을 때 처리 ★ CHAT-7 참조

사용자 입력에 도시·지역 정보가 없을 때 AI가 최대 2회 되묻기. 1회·2회는 동일 메시지를 표시하며 검색을 진행하지 않는다.

1·2회 되묻기 메시지 (동일):
[사용자] 조식 포함 호텔 추천해줘 [AI] 어느 도시 호텔을 찾으시나요? 아래에서 고르거나, 도시명을 직접 입력해 주세요. [제주] [부산] [서울] [오사카] [도쿄] [방콕] [싱가포르] [다낭] [파리] [뉴욕]
3회째 안내 메시지 (검색 진행 없음):
[AI] 아래에서 고르거나, 도시명을 직접 입력해 주세요. [제주] [부산] [서울] [오사카] [도쿄] [방콕] [싱가포르] [다낭] [파리] [뉴욕]
  • 되묻기 최대 횟수: 2회. 3회째 입력에도 지역 없으면 검색 없이 안내 메시지만 표시
  • 인기 도시 뱃지 10개 고정: 제주·부산·서울·오사카·도쿄·방콕·싱가포르·다낭·파리·뉴욕
  • 도시 뱃지 탭 또는 직접 입력 시: 이전 대화에서 쌓인 조건(조식·가격대·성급 등)을 합산하여 KAYAK API 재호출
  • 도시 직접 입력도 뱃지 탭과 동일하게 처리
8-3 범위 외 질문 처리
[AI] 저는 호텔 검색만 도와드릴 수 있어요. 어느 도시 호텔을 찾아드릴까요?
8-4 목적지 변경 시 누적 조건 초기화 ★ CHAT-7 케이스 D 참조

대화 중 목적지(도시·지역)가 변경되면 이전 대화에서 쌓인 조건 전체를 초기화하고 새 목적지 기본값으로 재검색한다.

[사용자] 서울 강남 조식 포함 호텔 찾아줘 [AI] 서울 강남에서 조식 포함 호텔 찾아봤어요. (결과 3개) [사용자] 오사카로 바꿔줘 [AI] 오사카 호텔을 새로 찾아드릴게요. ⚠ 목적지가 바뀌어서 이전 조건(조식 포함)은 초기화됐어요. 오사카 기본 조건으로 검색해요. [결과 3개]
  • 초기화 범위: 목적지 외 조건 전체 (조식·가격대·성급·숙소유형 등) 초기화
  • 초기화하지 않는 경우: 날짜·인원 등 목적지 외 조건만 변경 시 기존 누적 조건 유지
    • 예: "날짜를 다음 주로 바꿔줘" → 조식 포함 조건 유지
  • UI 안내: 초기화 시 응답 문장에 어떤 조건이 사라졌는지 명시 (사용자 혼란 방지)

제11조

AI 필터 (스마트 필터) v0.2 신설

11-1 위치 및 진입

검색결과(S01) 필터바 맨 앞에 [✦ AI 필터] 칩을 배치한다. 기존 필터칩([⊟ 전체][인기순↕][1박 가격][성급][평점 7.0+]) 앞에 위치.

[✦ AI 필터] [⊟ 전체] [인기순↕] [1박 가격] [성급] [평점 7.0+]
  • 칩 스타일: 파란 테두리 + 펄스 애니메이션(눈에 띄게)
  • 탭 → 자연어 입력 바텀시트 오픈 ("AI가 답변합니다 · 원하는 조건을 자유롭게 입력하세요")
  • AI 필터는 현재 검색결과 내 필터 전용 (지역·날짜·인원 변경 → 챗봇 인계)
11-2 입력 분기 (결과 내 필터 vs 챗봇 인계) ★ 개발자 필독

입력 문장을 LLM이 파싱해 '결과 내 필터' 또는 '챗봇 인계(새 검색)'로 분기한다.

구분대상 조건동작
결과 내 필터 성급 · 가격대 · 숙소유형 · 편의시설(수영장·조식 등) · 평점 · 브랜드 현재 검색(지역·날짜·인원) 유지한 채 결과에 필터만 적용 → 하단 토스트 "✓ 필터가 적용되었습니다"
챗봇 인계 (새 검색) 지역 · 체크인/아웃 · 인원 변경 / 필터 컬럼·값과 무관한 입력("조용한 곳" 등) 채팅창으로 인계 → 챗봇이 새로 검색

분기 예시:

  • "수영장 있는 5성급", "1박 20만원 이하" → 결과 내 필터
  • "오사카로 바꿔줘", "다음주로", "조용한 곳" → 챗봇 인계
