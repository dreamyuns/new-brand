AUTH-A 주석

1
GNB [로그인] 버튼 탭 → 바텀시트 진입
  • S01/S02 GNB의 [로그인] 버튼 탭 시 이 AUTH 바텀시트 슬라이드업.
  • S01·M01 랜딩형 GNB에서는 비로그인 상태일 때 [로그인] 버튼 표시.
  • 로그인 완료 후 GNB에서 [로그인] 버튼 미노출, [☰] 전체메뉴 버튼만 표시.
정책서 5-1절 / 8-1절.
진입 경로 1: GNB [로그인] 버튼 탭
진입 경로 2: 아웃링크 로그인 유도 모달(MODAL-A/B) [회원가입/로그인] 버튼 탭
2
통합 바텀시트 — 로그인/회원가입 1화면
  • 로그인과 회원가입을 별도 화면으로 분리하지 않고 단일 바텀시트로 통합.
  • 이메일 입력 → 인증코드 확인 → 서버에서 기존/신규 회원 자동 판별.
  • 기존 회원: 바로 로그인 완료.
  • 신규 회원: 약관 동의 단계(AUTH-E) 추가.
  • 바텀시트는 배경(이전 화면) 위에 오버레이로 표시.
정책서 8-1절 — 통합 바텀시트.
3
서브카피 — 고정 하드코딩 문구
  • 고정 문구: "로그인하고 캐시백 혜택을 받아보세요"
  • 진입 경로 무관하게 항상 동일 문구 표시. AUTH-A 진입 경로는 사실상 단일화 (비로그인 상태에서 공급사 아웃링크 버튼 탭 → MODAL-A [회원가입/로그인] 탭 → AUTH-A).
정책서 8-1절.
4
이메일 입력 필드
  • 이메일 주소만 입력. 비밀번호 필드 없음 — 이메일 인증코드 방식으로 신원 확인.
  • 가입 시 이메일이 유일한 필수 수집 정보.
  • 이름·성·국적·전화번호는 가입 완료 후 계정관리에서 선택 입력.
정책서 8-2절 — 비밀번호 없는 이메일 인증코드 방식 / 8-9절 — 이메일만 필수 수집.
5
[인증코드 받기] — 비활성(disabled)
  • 이메일 미입력 또는 유효하지 않은 형식(@ 미포함, 도메인 없음 등) 상태에서 버튼 비활성(회색 처리). 탭 불가.
  • 유효한 이메일 형식 입력 완료 시 즉시 활성(인디고 버튼) 전환 — AUTH-B 상태.
정책서 8-2절.
활성 조건: @ 포함 유효한 이메일 형식 입력 완료 (실시간 클라이언트 검증)
6
SNS 로그인 — MVP Phase 2 (UI만 표시)
  • 애플·구글 버튼 UI 위치만 표시. MVP에서 실제 OAuth 연동 동작 없음.
  • 탭 시 "서비스 준비 중입니다" 토스트 메시지 표시.
  • Phase 2에서 애플·구글·카카오·네이버 순으로 구현 예정.
  • 버튼에 "Phase2" 뱃지 표시로 미완성 상태 명시.
정책서 8-10절 — Phase 2 예정 (애플·구글·카카오·네이버).
7
약관 동의 사전 고지 문구
  • [인증코드 받기] 또는 SNS 버튼 탭 시 약관에 동의한 것으로 간주한다는 사전 고지 텍스트.
  • 이용약관·개인정보처리방침 링크를 포함하여 탭 시 해당 약관 전문 표시.
  • 신규 회원: 인증 완료 후 약관 동의 화면(AUTH-E)에서 한 번 더 명시적 동의 수집.

1
← 이메일 변경
  • 이메일 주소를 잘못 입력했거나 변경이 필요한 경우 AUTH-B(이메일 입력 활성) 단계로 이전.

  • 이미 발송된 인증코드는 무효가 됨.

2
인증코드 입력 화면 — 6자리 박스 UI
  • AUTH-C 발송 성공 후 이 화면으로 자동 전환(슬라이드).
  • 6자리 인증코드를 각 자리별 독립 박스(6개)에 입력하는 UI.
  • 각 박스에 1자리 숫자 입력 시 자동으로 다음 박스로 포커스 이동.
  • 붙여넣기(paste) 시 6자리 자동 분배 지원.
정책서 8-3절 — 6자리 박스 입력 UI.
3
발송 이메일 주소 표시

  • 주소가 잘못됐다면 [← 이메일 변경] 탭으로 수정 가능.
정책서 8-2절.
4
코드 재전송
  • 인증코드를 받지 못했거나 유효시간이 만료된 경우 탭하여 새 코드 재발송.
  • 재전송 횟수 제한 없음.
  • 탭 시 새 코드 발송 API 호출 + 타이머 5:00 리셋 + 발송 중 로딩 처리(버튼 일시 비활성).
  • 이전 코드는 즉시 무효.
정책서 8-3절 — 재전송 횟수 제한 없음.
5
유효시간 타이머
  • 코드 발송 시점부터 5분(5:00) 카운트다운.
  • 붉은색(#ef4444) 강조.
  • 유효시간 내 6자리 입력 완료 및 확인 필요.
정책서 8-3절 — 유효시간 5분.
0:00 도달: 코드 박스 비활성(입력 잠금) + "유효시간이 만료됐습니다. 코드를 재전송해 주세요." 안내 + [코드 재전송] 강조
6
[확인] 버튼 — 6자리 미완성 시 비활성 (폴백)

  • [확인] 버튼은 자동 확인이 작동하지 않는 경우를 대비한 폴백(fallback) 수동 제출 버튼.
  • 6자리 미완성 상태에서 비활성(회색) 표시.
