1
GNB — 브랜드 로고
  • 상단 sticky 고정(z-index:30).
  • 로고 탭 시 → M01 메인으로 이동.
  • S01은 랜딩형 GNB이므로 뒤로가기 버튼 없음.
  • 로고는 항상 표시.
sticky top:0, z-index:30. M01 GNB와 동일 컴포넌트.
2
GNB — [로그인] · [☰ 전체메뉴]
  • 비로그인 상태: [로그인] 버튼(primary, 파란색) + [☰] 버튼 동시 노출.
  • 로그인 완료 상태: [로그인] 버튼 미노출, [☰] 버튼만 표시.
  • [로그인] 탭: 로그인/회원가입 통합 바텀시트(AUTH) 진입.
  • [☰ 전체메뉴] 탭: MY01 마이페이지 전체화면으로 직접 이동 — 드롭다운 없음.
M01 주석 2번과 동일 정책.
3
재검색바 (요약형 · 전체 탭) ★
  • GNB 바로 아래 sticky 고정(top:45px). 목적지·날짜·인원 한 줄로 요약 표시.
  • 표시 형식 예: "서울 · 6/18(목) → 6/19(금) · 성인 2명"
  • 전체 행 탭 시 → GNB는 고정 상태 유지, GNB 아래에 검색폼이 슬라이드다운(S01-A). 검색폼 아래 결과 영역은 반투명 딤 처리.
  • 딤 탭 또는 [검색하기] 실행 시 검색폼 슬라이드업으로 닫힘.
  • [수정] 별도 버튼 없음 — 전체 행 탭이 유일한 진입 방법.
검색결과 갱신 트리거 3가지 (정책서 5-10절 v0.28)
  • ① 검색 조건 변경 (날짜·인원·목적지 수정 후 [검색])
  • ② 필터·정렬 변경
  • ③ 재검색바 탭 → 검색폼 → [검색하기]
  • ※ TTL 기반 자동 갱신·스크롤 위치 복원 — MVP 미적용 (v0.28 삭제)
sticky. GNB 높이(45px) 아래 고정. [수정] 버튼 없음 — 전체 행 탭으로 진입.
4
⊟ 전체 필터 칩
  • 탭 시 S01-D 전체 필터 바텀시트 슬라이드업.
  • 뱃지 숫자: 현재 적용된 필터 개수(가격범위·성급·숙소유형·브랜드체인·평점7.0+ 각 1개씩 카운트). 0이면 뱃지 미표시.
  • 활성 상태(파란 강조): 1개 이상 필터 적용 시. 비활성 상태: 필터 없을 때.
← API: minPrice / maxPrice / starRating / propertyTypes / chains / guestRatings
5
정렬 칩
  • 탭하면 아래에서 정렬 선택 시트가 올라옵니다. (→ S01-E)
  • 칩에는 지금 선택된 정렬 기준이 표시됩니다. 기본값: "인기순 ↕"
  • 선택 가능한 정렬 기준 4가지:
  • • 인기순 — 예약이 많고 반응 좋은 순서
  • • 낮은 가격순 — 제일 저렴한 숙소부터
  • • 평점 높은순 — 투숙객 평점이 높은 순서
  • • 거리순 — 검색한 위치에서 가까운 순서
  • 거리순 주의:
  • → 지도 뷰에서 [이 지역 검색]을 사용했을 때(지도 화면 범위로 검색한 경우)만 정확하게 동작합니다. 정렬은 서버가 자동으로 처리합니다.
  • → 도시명(예: "서울")으로 검색한 경우 거리순을 선택하면 어떻게 동작하는지 아직 확인이 필요합니다. ⚠️ 개발 시 실제 테스트 후 처리 방법 결정 필요.
← API sortField: popularity / minRate / consumerRating / distance
6
개별 필터 칩 (가격·성급·숙소유형·브랜드체인·평점 7.0+)
  • 각 칩 탭 시 해당 항목만의 개별 바텀시트 진입 (S01-D-1 1박가격 / D-2 성급 / D-3 숙소유형 / D-4 브랜드·체인 / D-5 평점 7.0+).
  • 활성 상태: 해당 필터가 1개 이상 적용된 경우 칩 파란 강조(eef2ff 배경, #4f46e5 텍스트·테두리).
  • 비활성 상태: 흰 배경, 회색 테두리.
  • 필터바는 가로 스크롤 가능(overflow-x: auto). ⊟ 전체 칩이 항상 가장 좌측 고정.
확정: 2026-06-16 / 정책서 v0.29 5-6절.
7
결과 수 헤더
  • 도시 검색: "{지역명} · 총 {N}개 숙소" 형식 (예: "서울 · 총 843개 숙소").
  • 숙소명 검색(5-9절): 해당 숙소의 소속 도시명 기준으로 헤더 표시. 검색한 숙소를 카드 목록 최상단 고정, 이후 동일 도시의 나머지 호텔 나열.
  • 서브텍스트: 체크인~체크아웃 날짜 + 인원 + 박수 (예: "6월 18일(목) — 19일(금) · 성인 2명 · 1박").
  • 필터 적용 시 숫자는 필터 결과 수로 실시간 반영.
← totalResults 또는 results[].length
8
스켈레톤 (Progressive Loading)
  • 검색 API 호출 즉시 스켈레톤 카드(shimmer 애니메이션) 표시. onlyIfComplete=false — 부분 응답 수신 중에도 카드 순차 교체.
  • 초기 로딩: 스켈레톤 카드 3개 노출(이미지 영역 + 텍스트 라인 + 공급사 영역 + 더보기 영역 shimmer).
  • [목록 더보기] 버튼 로딩 중: 기존 카드 아래 스켈레톤 2~3개 추가 표시.
  • 완료 시: 스켈레톤 → 실제 카드로 교체(fade-in). 응답 데이터 없으면 빈 상태 처리.
정책 5-8절 (v0.29). 초기 pageSize=50.

카드 이미지

  • 이미지 소스: images[0] (Static Data Feeds 캐시 기준, KAYAK 제공 호텔 사진).
  • 로딩 전: 회색(#ebebeb) 플레이스홀더.
  • 이미지 에러·없음 시: 기본 대체 이미지(숙소 아이콘 또는 브랜드 이미지) 표시.
← images[0] (Static Data Feeds 캐시 기준)

이미지+호텔정보 클릭 정책 ★
  • 카드 상단 전체 영역(이미지 + 호텔명·성급·위치·평점 영역) 탭 → S02 숙소상세 화면으로 이동.
  • [이동→] 버튼과 동작 완전 분리: 상단 탭은 S02, [이동→] 탭은 아웃링크(OUT01) 흐름.
  • 로그인 여부 관계없이 S02 진입 허용 (비로그인도 숙소 상세 조회 가능).
확정: 2026-06-11.
11
호텔명 / 성급 / 위치 / 평점
  • 호텔명: translatedName(ko_KR) 우선 → 없으면 name 폴백. 1줄 말줄임(text-overflow:ellipsis).
  • 성급: starRating 값을 ★ 개수로 변환 (1~5). null이면 미표시.
  • 위치: address.neighborhood 우선 → 없으면 address.city. "📍 {지역명}" 형식.
  • 게스트 평점: guestRating 수치만 표시. 레이블(완벽·매우좋음 등) 미표시 — S02 숙소상세에서 sentiment 파라미터 값으로 텍스트 표시. 파란 배지(#4f46e5)로 표시.
  • 리뷰 수: numberOfReviews — "(N개)" 형식. 0 또는 null이면 미표시.
12
공급사 행 — topRates 최저가 1개 ★
  • 전체 공급사 중 totalRate(세금·봉사료 포함 최종가) 기준 최솟값 1개만 표시.
  • 표시 항목: 공급사 로고(favicon 또는 컬러 약자 박스) + 공급사명 + 총금액 + 박수 기준 텍스트 + [이동→] 버튼.
  • 가격 형식: ₩{totalRate} / {N}박 기준 (선택 통화에 따라 $, ¥ 등으로 변환).
  • 동일 최저가 공급사 다수: providers 배열 첫 번째 항목 사용.
  • v0.19 정정: topRates = KAYAK 추천 알고리즘 기반 상위 subset (최저가 순서 보장 아님). 카드 최저가 표시는 topRates 내 isCheapestRate: true 필드 기준. topRates는 응답 크기 절감 목적.
← providers[providerIndex].name / results[].totalRate (세금 포함)
확정: 2026-06-11. v0.19 정정(2026-06-12): topRates = KAYAK 추천 알고리즘 subset (최저가 보장 아님). 최저가 식별: isCheapestRate: true 기준. 정책서 5-3절·12-1절.
13
[이동→] 버튼 — 아웃링크
  • 비로그인 상태 탭: 캐시백 N% 강조 로그인 유도 모달 표시 → [회원가입/로그인] 탭 시 AUTH 바텀시트 → 완료 후 OUT01 이동 / [건너뛰기] 탭 시 바로 OUT01 이동.
  • 로그인 상태 탭: 로그인 유도 모달 없이 바로 OUT01(이동 중 화면) → 타 OTA로 이동.
  • 딥링크 URL: results[].bookUri (KAYAK CPA 어필리에이트 딥링크). 타사 예약 완료 시 CPA 수익 발생.
← results[].bookUri (KAYAK CPA 딥링크)
14
무료취소 / 나중에 결제 배지
  • 무료취소: hasFreeCancellation=true → "✅ 무료취소" 초록 배지(#d1fae5 배경, #065f46 텍스트) / false → "❌ 환불불가" 빨간 배지(#fee2e2 배경, #991b1b 텍스트).
  • 나중에 결제: canPayLater=true → "💳 나중에 결제" 파란 배지(#e0e7ff 배경, #3730a3 텍스트) / false 또는 null → 배지 미표시.
  • 배지는 공급사 행 바로 아래(.pv-badges 영역)에 나란히 표시.
← results[].hasFreeCancellation / results[].canPayLater = true 시 각 배지 표시. false 또는 null 미표시.
15
💸 캐시백 최대 N% 배지 ★
  • 표시 조건: 최저가 공급사의 cashback.type 값에 따라 분기.
  • PERCENTAGE → "💸 캐시백 최대 N%" (N = cashback.value, 소수점 첫째 자리까지 표시)
  • FLAT → "💸 캐시백 최대 ₩N" (N = cashback.value, 선택 통화 기호로 변환)
  • NONE 또는 cashback 필드 없음 → 배지 미표시
  • 표시 대상: 타 OTA 아웃링크 공급사만 (올마이투어 자사 상품은 MVP 제외 — Phase 2).
  • 회원 구분 없음: 비로그인·로그인 동일하게 배지 표시. 로그인 유도는 [이동→] 탭 시점에만 적용.
  • 데이터 출처: KAYAK providers[].cashback 데이터 그대로 반영 (자사 재계산 없음). 방식 B(고객 직접 지급)는 Phase 2 — MVP는 배지 표시만 운영.
← providers[providerIndex].cashback.value (%) 기준
확정: 2026-06-11 / 정책서 9-7절·9-6절. 방식 2 Phase 2.
16
N개 가격비교 더보기 ★
  • 카드 하단 고정 영역. 표시 문구: "{numberOfProviders}개 가격비교 더보기 ›".
  • 탭 동작: S02 숙소상세 화면으로 이동 + 가격비교 탭 기본 활성 상태로 진입.
  • numberOfProviders = 0 또는 1: "더보기" 행 미표시 (공급사 1개면 추가 비교 없음).
  • 로그인 여부 관계없이 S02 진입 허용.
← numberOfProviders 기반 숫자
확정: 2026-06-11.
17
식사 포함 배지 (inclusions)
