화면 주석 — M01-A

1
브랜드 로고
  • 서비스명 "allmy meta" 텍스트 로고.
  • 탭 시 메인 화면(M01-A) 새로고침(스크롤 최상단 복귀 포함).
  • sticky GNB 내 좌측 고정 배치.
  • GNB는 모든 화면 최상단에 position: sticky; z-index 최상위로 고정.
리브랜딩 확정 전까지 "allmy meta" 임시 사용. 로고 디자인은 디자이너 작업.
2
[로그인] + [☰ 전체메뉴] 버튼 — 로그인 상태별
  • 비로그인 상태: [로그인] 버튼(primary, 보라색) + [☰] 버튼 동시 노출. [로그인] 탭 시 로그인 바텀시트(auth_login.html) 진입.
  • 로그인 완료 상태: [로그인] 버튼 미노출. [☰] 버튼만 표시.
  • [☰ 전체메뉴] 탭 동작: 별도 드롭다운 없이 마이페이지(MY01) 전체화면으로 직접 이동.
정책서 5-1절. GNB 랜딩형(M01·S01) 패턴: 로고 + [로그인] + [☰]. 세부화면(S02·AUTH)은 ← 뒤로가기 + [로그인] + [☰]. MY01~MY03(마이페이지 하위)은 ← 뒤로가기만 표시.
MY01 화면 구성 상세: 비로그인 = [회원가입/로그인] 버튼 + 언어·통화·약관 메뉴 / 로그인 = 프로필(이름+이메일) + 계정관리·캐시백 내역·언어·통화·약관 + [로그아웃].
3
목적지 입력 필드
  • 기본 상태: 플레이스홀더 "어디로 가시나요?" 표시.
  • 탭 시 M01-B(자동완성) 전체화면으로 전환.
  • M01-B에서 항목 선택 후 복귀하면 선택된 지명이 이 필드에 표시됨.
  • 목적지 미선택 상태에서 [검색] 탭 시 이 필드에 포커스 이동 + 토스트 에러 "목적지를 입력해주세요." 표시.
← Autocomplete API 선택값: fullname (표시용 지명) / entityKey (검색 파라미터 destination 값으로 사용)
정책서 4-1절. 목적지 미선택 시 [검색] 버튼 비활성 또는 탭 시 유효성 오류 처리. 둘 중 하나로 구현 (개발팀 결정).
4
체크인 날짜
  • 탭 시 M01-C(날짜 선택 캘린더) 전체화면으로 전환.
  • 기본값: 오늘 날짜 + 7일 (앱 실행 시점 기준 자동 계산).
  • 표시 형식: "M월 D일 (요일)" (예: 6월 18일 (목)).
← API 파라미터: checkIn=YYYY-MM-DD
정책서 4-7절. 오늘 이전 날짜 선택 불가 (캘린더에서 비활성 처리). 체크인 변경 시 체크아웃이 체크인보다 같거나 이전이면 체크아웃 자동 재조정 (체크인 + 1일).
5
체크아웃 날짜
  • 탭 시 M01-C(날짜 선택 캘린더) 전체화면으로 전환.
  • 기본값: 체크인 날짜 + 1일.
  • 표시 형식: "M월 D일 (요일)" (예: 6월 19일 (금)).
  • 박수(nights)는 필드에 직접 표시하지 않고 캘린더 하단 선택 정보 바에서만 표시.
← API 파라미터: checkOut=YYYY-MM-DD
정책서 4-7절. 체크아웃 > 체크인 강제. 체크아웃 = 체크인 선택 시 자동으로 체크인 + 1일로 조정. 체크아웃 < 체크인인 날짜는 캘린더에서 선택 불가 처리. 세금 포함 최종가 기준(includeTaxesInTotal=true, includeLocalTaxesInTotal=true)으로 검색 파라미터 전송.
6
인원 · 객실 필드
  • 탭 시 M01-D(인원·객실 설정) 전체화면으로 전환.
  • 기본값: 객실 1개 / 성인 2명 / 아동 0명.
  • 표시 형식: "성인 N명 · 객실 N개" (아동 있을 경우: "성인 N명 · 아동 N명 · 객실 N개").
  • M01-D에서 [검색] 탭 시 값 확정 후 이 필드 업데이트.
← API 파라미터: rooms. 단일객실 성인2="2", 성인2+아동(8세)="2:8", 다객실(성인2 | 성인1+아동5세)="2|1:5"
정책서 4-9절. 객실당 성인 최소 1명. 객실당 아동 최대 4명(만 0~17세). 객실당 총 인원(성인+아동) 최대 8명. 최대 8객실.
7
[검색] 버튼
  • 목적지 선택 필수.
  • 날짜·인원은 기본값이 자동 적용되므로 별도 선택 없이도 탭 가능.
  • 탭 시 입력값을 로컬 스토리지에 최근 검색으로 저장 후 S01(검색결과) 화면으로 이동.
  • 목적지 미선택 시 S01로 이동하지 않고 유효성 오류 처리.
← GET /api/affiliate/search/v1/hotels 호출. 파라미터: destination(entityKey), checkIn, checkOut, rooms, currencyCode=KRW, languageCode=ko_KR, includeTaxesInTotal=true, includeLocalTaxesInTotal=true, sortField=popularity
정책서 4-7절. [검색] 탭 시 최근 검색 로컬 스토리지 저장: 최대 10개, 초과 시 가장 오래된 항목 삭제. 저장 항목: 목적지명 + entityKey + 체크인·체크아웃 날짜 + rooms 파라미터.
8
최근 검색 섹션
  • 로컬 스토리지 기반 최근 검색 이력 표시.
  • 저장 항목: 목적지명 + entityKey + 체크인·체크아웃 날짜 + rooms 파라미터(인원 구성 포함).
  • 각 항목 탭 시 해당 조건이 검색폼(목적지·날짜·인원)에 그대로 복원됨 — 복원 후 자동 검색하지 않고 사용자가 [검색] 버튼을 직접 탭해야 S01로 이동.
  • 최근 검색이 0건일 경우 이 섹션 미표시.
정책서 4-8절. 최대 10개. 최근 검색 순 정렬(최신이 최상단). 로컬 스토리지 저장으로 서버 동기화 없음(MVP). 기기 변경 시 초기화 (Phase 2 서버 동기화 검토).
9
최근 검색 개별 삭제 (✕)
  • 각 최근 검색 항목 우측의 ✕ 버튼.
  • 탭 시 해당 항목만 로컬 스토리지에서 즉시 삭제하고 UI에서 제거.
  • 삭제 확인 다이얼로그 없음(즉시 처리).
  • 전체 삭제는 M01-B(자동완성) 화면의 [전체 삭제] 버튼 사용.
정책서 4-8절. 개별 삭제 후 남은 항목은 순서 유지. 0건이 되면 최근 검색 섹션 전체 미표시.
10
추천 호텔 가로 스크롤
  • 가로 스크롤 카드 형태로 인기 호텔 노출.
  • 각 카드 탭 시 해당 호텔의 S02(숙소 상세) 화면으로 이동 (현재 검색폼 날짜·인원 조건 유지).
  • 카드 표시 항목: 호텔 대표 이미지 / 호텔명 / 위치(도시·지역) / 원가(취소선) / 최저가(KRW).
  • MVP에서는 어드민에서 직접 지정한 호텔 목록 고정 노출 방식 사용.
MVP: 어드민 지정 호텔 목록 정적 노출. 실시간 가격 연동은 Phase 2 검토. 호텔 이미지는 hotelfeed 썸네일 URL 사용.
카드 정보: 호텔명 / 위치(도시·지역명) / 원가(취소선 표시) / 최저가. "모두 보기 →" 탭 동작은 MVP 미결 — 기본 검색폼 조건으로 S01 이동 또는 미구현 중 결정 필요.
M01-B목적지 자동완성 — 목적지 필드 탭 후 전환
전체 화면 오버레이. 완전한 음절 1자 이상 입력 시 실시간 자동완성 결과 표시.
노출 순서: city 최상단 고정 → neighborhood/region → hotel/airport 등 (클라이언트 정렬). 최대 6개 노출.
항목 선택 시: 검색폼 목적지 필드에 입력됨 → [검색] 버튼 탭 시 S01 이동 (직접 이동 없음)
⚙ 공통 정책 — M01-B 자동완성
  • API: GET /api/affiliate/autocomplete/v1/hotels (4-1절)
  • 트리거: 완전한 음절 1자 이상. 자음·모음 단독 입력 트리거 안 함 (4-1절)
  • 최대 결과: 6개. 클라이언트 정렬: city → region → hotel/resort/기타 순 (4-1·4-2절)
  • 표시 형식: 도시=fullname 1행. 호텔=hotelName+places[] 2행. displayPlaceType 배지 미표시 (4-3절)
  • 아이콘: API 응답에 없음 → primaryPlaceType 기반 자체 아이콘 세트 필요 (4-6절)
  • 결과 없음: "검색 결과가 없습니다" 드롭다운 표시 (4-6절)
  • 포커스 + 빈 입력: 드롭다운 없음. 최근검색 있으면 최근검색 표시 (4-6·4-8절)
  • 현위치 조회: MVP 제외, UI 미노출 (4-6절)
기준: v0.29 4-1절~4-6절·4-8절

화면 주석 — M01-B

1
뒤로가기 (←)
  • 탭 시 M01-A(메인 홈)으로 복귀.
  • 현재 자동완성 입력값(검색어)은 초기화됨.
  • 단, 검색폼의 목적지 필드는 M01-B 진입 이전에 이미 선택된 값이 있었다면 그 값을 그대로 유지 (새로 선택 시도 중 취소한 경우).
  • 자동완성 드롭다운 닫힘.
정책서 4-1절.
2
입력 박스 · 클리어(✕) · 트리거 조건
  • 트리거 조건: 완전한 음절 1자 이상 입력 시 API 호출 및 드롭다운 표시. 자음·모음 단독(ㅅ, ㅓ 등) 입력은 API 호출 트리거 안 함.
  • 포커스 후 빈 입력: 드롭다운 미표시. 최근 검색 섹션(cn7)만 노출.
  • 디바운스: 300ms 적용 권장 (입력 중 불필요한 API 과호출 방지).
  • ✕ 탭: 입력값만 지워지고 드롭다운 닫힘. 포커스는 입력 박스에 유지.
  • 결과 없음: "검색 결과가 없습니다" 드롭다운 표시.
← Autocomplete API: GET /api/affiliate/autocomplete/v1/hotels?query={입력값}&languageCode=ko_KR (최대 결과 6개 반환)
정책서 4-1~4-6절.
3
city 타입 결과 — 최상단 고정
  • API 반환 순서에 관계없이 primaryPlaceType = "city"인 결과를 항상 드롭다운 최상단에 배치 (클라이언트 정렬).
  • 표시: 1행만 — fullname 필드 사용 (예: "서울, 서울특별시, 대한민국").
  • displayPlaceType 배지(Hotel/Resort 등)는 MVP 미표시.
  • 항목 탭 시 해당 entityKey가 검색 파라미터 destination으로 설정되고 검색폼 목적지 필드에 지명 표시됨 — 자동 검색 없이 사용자가 [검색] 버튼을 직접 탭해야 S01로 이동.
← result[].primaryPlaceType 기반 클라이언트 정렬: city(1순위) → region/neighborhood(2순위) → hotel/resort/기타(3순위). 저장값: result[].entityKey
정책서 4-3~4-5절. 지명 언어: placefeed languageCode=ko로 한국어명 캐시 목표 — DB 구축 전까지 영어 지명 표시 허용.
API에서 city 타입이 반환되지 않은 경우, region → neighborhood → hotel 순으로 최상단 배치. 총 노출 개수는 API 반환값 전체(최대 6개).
4
neighborhood / region 타입 결과
  • city 다음 두 번째 우선순위로 표시.
  • 구·동·지역·행정구역 단위.
  • 표시: 1행만 — fullname 필드 사용 (예: "명동, 서울, 대한민국").
