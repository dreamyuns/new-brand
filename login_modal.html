화면 주석 — MODAL-A

진입
비로그인 [이동→] 탭 시 바텀시트 슬라이드업
  • 비로그인 상태에서 S01 검색결과 카드의 [이동→] 버튼 탭 시, 또는 S02 가격비교 탭의 [이동→] 버튼 탭 시 로그인 유도 모달 표시.
  • 딤(rgba 0,0,0,0.55) 오버레이 + 바텀시트 슬라이드업.
  • 배경 화면(S01/S02)은 블러 처리.
  • 캐시백 있는 공급사(cashback.type = PERCENTAGE 또는 FLAT): 이 화면(MODAL-A) 표시 — 캐시백 강조 박스 포함.
  • 캐시백 없는 공급사(cashback.type = NONE): MODAL-B 표시 — 캐시백 강조 박스 없이 기본 로그인 유도만.
  • 로그인 상태인 경우 이 모달 없이 OUT01로 직접 진입.
정책서 7-1절 / 9-7절.
1
[✕] 닫기 버튼
  • 탭 시 모달 슬라이드다운 닫힘.
  • S01(검색결과) 또는 S02(숙소 상세) 화면으로 복귀.
  • 아웃링크 미진행.
  • 사용자가 검색결과를 계속 탐색할 수 있도록 상태 유지.
정책서 7-1절.
2
캐시백 강조 박스
  • 모달 상단 보라색 그라데이션(#7c3aed → #4f46e5) 박스.
  • 캐시백 혜택을 첫 시선에 강조하여 로그인 동기를 부여하는 핵심 UI.
  • MODAL-A에서만 표시 (캐시백 있는 공급사 전용).
  • cashback.type = NONE이면 이 박스 자체 미렌더링 → MODAL-B로 분기.
캐시백 강조 방식 확정 (아웃링크 직전 모달에서만 안내) — 정책서 9-7절·9-6절. 데이터: KAYAK providers[].cashback 그대로 반영. 방식 2(실지급) Phase 2. 2026-06-11 확정.
3
캐시백 N% 실시간 표시
  • 해당 공급사의 providers[selectedProviderIndex].cashback.value를 기반으로 강조 박스 내 캐시백 문구를 동적 표시.
  • 비회원과 로그인 회원 모두 동일하게 표시 (로그인 유도 목적이므로 비회원 상태에서 표시).
← providers[selectedProviderIndex].cashback.type / .value
정책서 9-7절.
◼ cashback.type = PERCENTAGE: "최대 N% 적립" (N = cashback.value)
◼ cashback.type = FLAT: "최대 ₩N 적립" (N = cashback.value, 원 단위 표기)
◼ cashback.type = NONE: MODAL-B로 분기 (이 강조 박스 없음)
4
호텔 정보 요약 행
  • 사용자가 탭한 호텔의 컨텍스트를 유지하여 "어떤 호텔·공급사에 대한 캐시백인지" 즉시 인지하도록 표시.
  • 항목: 대표 이미지 썸네일 + 호텔명 + 공급사명 + 체크인·아웃 날짜 + 가격.
  • S01/S02에서 이미 로드된 데이터 재사용 (API 재호출 없음).
← S01/S02 카드 데이터 재활용 (hotelName, providerName, checkIn, checkOut, totalRate)
정책서 7-1절.
5
[회원가입/로그인] 버튼
  • 탭 시 이 모달(login_modal) 슬라이드다운 → AUTH 로그인/회원가입 통합 바텀시트(AUTH-A) 슬라이드업.
  • 사용자가 탭했던 공급사의 bookUri, selectedProviderIndex 등 아웃링크 진행에 필요한 state를 세션에 보존.
  • 로그인/회원가입 완료 후 보존된 state로 OUT01-A 자동 진입.
정책서 7-1절 (버튼 단일화) / 8-1절 (로그인·회원가입 통합 바텀시트).
6
[이 혜택 없이 계속하기 →] 건너뛰기
  • 로그인 없이 비회원 상태로 OUT01-A로 직접 이동.
  • userTrackId는 생성되어 bookUri에 포함되므로 CPA 추적은 정상 작동.
  • 단, 회원 계정 연결 없이 진행하므로 어썸캐시 귀속 불가.
  • CPA 수익(자사 60% : KAYAK 40%)은 발생.
  • 강제 로그인 차단 금지 원칙에 따라 항상 노출.
강제 로그인 금지 원칙 / 건너뛰기 항상 노출 — 정책서 7-3절.

화면 주석 — MODAL-B

차이
MODAL-A vs MODAL-B 차이점
  • cashback.type = NONE인 공급사 탭 시 캐시백 강조 박스를 렌더링하지 않고 이 MODAL-B 표시.
  • 캐시백 수치를 내세울 근거가 없으므로 예약 내역 저장·향후 혜택을 간단히 안내하는 소프트 로그인 유도 방식.
  • 버튼 구조(회원가입/로그인 + 건너뛰기)와 호텔 정보 요약 행은 MODAL-A와 동일.
← providers[selectedProviderIndex].cashback.type = NONE → MODAL-B 표시
정책서 7-1절 / 9-7절.
1
기본 로그인 유도 문구
  • "로그인 후 예약하세요" 제목 + "로그인하면 예약 내역을 저장하고 향후 캐시백 혜택을 받을 수 있어요" 설명.
  • 캐시백 수치 없이 예약 내역 저장과 미래 혜택 가능성으로 소프트 유도.
  • 사용자에게 강압감 없이 로그인을 권유.
정책서 7-1절.
2
건너뛰기
  • MODAL-A의 건너뛰기와 동일 동작.
  • 비로그인 상태로 OUT01-A → 타 OTA 이동.
  • userTrackId 포함 CPA 추적 유지, 어썸캐시 귀속 불가.
  • 강제 로그인 금지 원칙에 따라 항상 노출.
