ERR-A404 — 페이지를 찾을 수 없음
존재하지 않는 URL 접근, 삭제된 페이지, 오타 입력 등으로 해당 경로를 찾을 수 없을 때 표시.
GNB: [←](뒤로가기, 좌측) + 로고(중앙). 탭바 없음. [홈으로] 버튼 없음.
⚙ 공통 정책 — ERR-A 404
  • 노출 조건: 라우터에서 매칭 경로 없음 (404 Not Found)
  • GNB: 뒤로가기 버튼(좌측) + 로고(중앙). 로그인·메뉴 버튼 없음
  • 뒤로가기 버튼 동작: history.length > 1 → 직전 페이지 이동 / 이전 페이지 없음 → M01 메인으로 이동 (직접 URL 입력·외부 링크 진입 시)
  • 탭바: 미표시
  • [홈으로] 버튼: 미사용 (삭제). 사용자가 직접 판단해 뒤로가기로 복귀
  • 에러 코드 표기: 장식용 대형 텍스트 "404" 배경에 연하게 표시
⚠ 정책 미결 — 에러 페이지 라우팅 처리 방식 개발 확인 필요

화면 주석 — ERR-A (404 오류)

1
GNB — 로고 (가운데)
  • 로고는 GNB 가운데에 표시됩니다.
  • 로고 탭 시 → M01 메인으로 이동합니다.
탭 시 M01(메인)으로 이동.
2
GNB — 뒤로가기 버튼 (좌측)
  • GNB 오른쪽에 [←] 버튼을 표시합니다.
  • 탭 시 동작 2단계:
    1. 이전에 본 페이지가 있으면 → 바로 그 페이지로 돌아갑니다.
    2. 이전 페이지가 없으면 (앱을 처음 열자마자 오류 페이지로 왔거나, 주소창에 직접 입력한 경우) → M01 메인으로 이동합니다.
history.length > 1 → history.back() / 없으면 → M01 이동.

ERR-B일반 오류 — 서버·네트워크 오류
서버 오류(500), 네트워크 연결 실패, 예상치 못한 앱 오류 등 복구 방법을 안내할 수 없는 경우.
GNB: [←](뒤로가기, 좌측) + 로고(중앙). 탭바 없음. [홈으로] 버튼 없음.
⚙ 공통 정책 — ERR-B 일반 오류
  • 노출 조건: HTTP 5xx 오류, 네트워크 타임아웃, 앱 예외(Exception) 등 복구 불가 에러
  • 에러 코드 노출: 사용자에게 에러 코드(500 등) 직접 표시 않음. 개발자 콘솔·로그로만 확인
  • GNB: 뒤로가기 버튼(좌측) + 로고(중앙). 로그인·메뉴 버튼 없음
  • 뒤로가기 버튼 동작: ERR-A와 동일. history.length > 1 → 직전 페이지 이동 / 없으면 → M01 메인
  • 탭바: 미표시
  • [홈으로] 버튼: 미사용 (삭제)
  • ERR-A와 구분: 아이콘·문구만 다름. GNB·레이아웃 동일
⚠ 정책 미결 — 오류 유형별 세분화(네트워크/서버/앱) 개발 확인 필요

화면 주석 — ERR-B (일반 오류)

1
GNB — 로고 (가운데)
  • ERR-A와 동일. 로고 가운데 표시. 탭 시 → M01 메인으로 이동.
탭 시 M01(메인)으로 이동.
2
GNB — 뒤로가기 버튼 (좌측)
  • ERR-A와 동일한 동작. 이전 페이지가 있으면 그 페이지로 / 없으면 M01 메인으로 이동.
  • 오류 화면에서 굳이 홈으로 보내지 않고, 사용자가 있던 화면으로 그대로 돌려보내는 것이 더 자연스러운 흐름입니다.
history.length > 1 → history.back() / 없으면 → M01 이동.
3
에러 타이틀
  • 문구: "오류가 발생했어요"
  • 에러 코드(500 등)는 사용자에게 직접 노출하지 않습니다. 기술적인 코드보다 상황 설명이 더 친절하기 때문입니다.
4
에러 설명 문구
  • 문구: "잠시 후 다시 시작해 보세요."
  • 직접 코드에 넣어둔 고정 텍스트. 모든 일반 오류에 동일하게 표시합니다.
