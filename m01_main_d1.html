화면 주석 — M01_D1-A (비로그인)

1
GNB 좌측 — [로그인] 버튼
  • 비로그인 상태: 흰색 반투명 알약 형태 [로그인] 버튼 표시.
  • 탭 시 로그인 바텀시트(auth_login.html) 진입.
  • 로그인 완료 후: 아바타 원형 아이콘으로 전환 (B케이스 주석 참조).
⚠️ 미확정 — 로그인 후 좌측 영역(아바타 아이콘 vs 로고 등) 대표 논의 필요. 현재 B케이스에 아바타 임시 표현.
2
GNB 우측 — 캐시백 정보 버튼
  • 비로그인 또는 캐시백 0원 상태: "로그인 후 최대 혜택 적립" 문구를 버튼 형태로 표시.
  • 로그인 후 1원 이상 상태: "내 캐시백 / N,NNN원" 형태로 금액 표시 (B케이스 참조).
  • 탭 시: 로그인·비로그인 모두 적립 탭(하단 탭바 2번)으로 이동.
← 표시 금액: cashback.approved (적립완료 금액만)
정책서 9-1절. 적립완료(Approved) 금액만 표시. ⚠️ 대표 논의 필요 — 적립예정 합산 여부 최종 확인.
3
AI 검색바
  • "어디로 떠나세요?" 플레이스홀더가 있는 텍스트 입력창.
  • 탭 시 텍스트 직접 입력 가능.
  • 입력 후 확인(엔터 또는 돋보기 탭) 시 S01(검색결과) 이동.
  • ✦ 아이콘: AI 기능을 나타내는 시각 요소. 우측 [AI] 배지 함께 표시.
⚠️ 논의중 — AI 검색창 상세 기능(자연어 처리 방식, 파라미터 변환 등) 추후 확정 예정. 현재는 UI 위치·형태만 표현. 확정 후 주석 업데이트.
4
목적지 입력 필드
  • 기본 상태: "어디로 가시나요?" 플레이스홀더 표시.
  • 탭 시 자동완성 전체화면으로 전환 (기존 m01_main.html M01-B와 동일).
  • 목적지 미선택 시 [검색하기] 탭 → 목적지 필드에 포커스 + 토스트 "목적지를 입력해주세요."
← Autocomplete API: GET /api/affiliate/autocomplete/v1/hotels?query={입력값}&languageCode=ko_KR
정책서 4-1절. 자동완성 상세 동작은 기존 m01_main.html M01-B와 동일.
5
체크인 · 체크아웃 날짜
  • 각 필드 탭 시 날짜 선택 캘린더 전체화면 전환.
  • 기본값: 체크인=오늘+7일, 체크아웃=체크인+1일.
  • 표시 형식: "M월 D일 (요일)".
← API 파라미터: checkIn=YYYY-MM-DD, checkOut=YYYY-MM-DD
정책서 4-7절. 캘린더 상세 동작은 기존 m01_main.html M01-C와 동일.
6
인원 · 객실 필드
  • 탭 시 인원·객실 설정 전체화면 전환.
  • 기본값: 객실 1개 / 성인 2명 / 아동 0명.
  • 표시 형식: "성인 N명 · 객실 N개" (아동 있을 경우: "성인 N명 · 아동 N명 · 객실 N개").
← API 파라미터: rooms (예: 성인2 = "2", 성인2+아동8세 = "2:8")
정책서 4-9절. 인원설정 상세 동작은 기존 m01_main.html M01-D와 동일.
7
[검색하기] 버튼
  • 목적지 선택 필수. 날짜·인원은 기본값 자동 적용.
  • 탭 시 검색 조건을 로컬 스토리지에 최근 검색으로 저장 후 S01(검색결과) 화면으로 이동.
  • 목적지 미선택 시 S01 미이동 — 유효성 오류 처리.
← GET /api/affiliate/search/v1/hotels 호출. 파라미터: destination, checkIn, checkOut, rooms, currencyCode=KRW, languageCode=ko_KR, includeTaxesInTotal=true
정책서 4-7절.
8
하단 탭바 — [적립] 탭
  • 탭 시 적립 탭 전체화면으로 이동 (신규 화면, 별도 제작 예정).
  • 탭바는 1뎁스 화면(메인·적립탭)에만 표시됨. 검색결과(S01) 이후 화면에서는 미표시.
확정. 적립 탭 화면은 별도 파일로 신규 제작 예정.
9
하단 탭바 — [공작새/고객지원] 탭
  • 탭 시 서비스 FAQ 채팅창 화면으로 진입.
  • 채팅창에서 서비스 관련 질의응답 가능.
⚠️ 미확정 — 채팅창 상세 기능(AI 챗봇 vs 정적 FAQ 등) 대표 논의 필요. 확정 후 별도 화면기획서 제작. 현재는 탭 진입 동작만 표현.

화면 주석 — M01_D1-B (로그인 후)

1
GNB 좌측 — 로그인 후 (임시: 아바타 아이콘)
  • 로그인 완료 후 [로그인] 버튼 자리에 아바타 원형 아이콘으로 전환 (현재 임시 표현).
  • 마이페이지 진입점과 연계하여 최종 처리 방법 결정 필요.
⚠️ 미확정 — 선택지 3가지 대표 논의 필요:
A안: 아바타 아이콘 탭 → MY01(마이페이지) 이동
B안: 로고 표시 (마이페이지는 적립 탭 하단에 통합)
C안: 기타 방안
2
GNB 우측 — 캐시백 금액 표시
  • 로그인 후 캐시백이 1원 이상인 경우 "내 캐시백 / N,NNN원" 형태로 표시.
  • 표시 금액: 적립완료(Approved) 금액만 표시.
  • 탭 시 적립 탭(하단 탭바 2번)으로 이동.
  • 천 단위 구분 쉼표 표기 (예: 137,000원).
← 금액 참조: cashback.approved
2026-07-02 논의 — 적립완료(Approved)만 표시. ⚠️ 대표 논의 필요 — 적립예정 합산 여부 최종 확인.
3
웰컴 메시지
  • 로그인 후에만 표시되는 인사 문구.
  • 표시 형식: "반가워요 👋" (이름 미노출).
  • 서브 텍스트: "오늘도 최저가 호텔을 찾아드릴게요" (고정 문구).
사용자 이름 미노출 — 이름 데이터 미입력 가능성 및 개인정보 노출 최소화 방침.
4
검색폼 — 이전 검색 조건 복원
  • 로그인 후 메인 진입 시, 마지막 검색 조건(목적지·날짜·인원)이 검색폼에 복원되어 표시.
  • 복원 데이터 출처: 로컬 스토리지의 최근 검색 1번째 항목.
  • 최근 검색이 없는 경우: 기본값(목적지 빈칸, 날짜 오늘+7일, 인원 성인2+객실1) 표시.
정책서 4-8절. 로컬 스토리지 기반으로 기기 변경 시 초기화.
