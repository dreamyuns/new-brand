  • featureSummary[]가 빈 배열이면 바텀시트 자체 미표시(버튼도 미노출).

S02-E 주석

1
총 평점
  • 단일 검색(GET /hotel) reviews 데이터의 guestRating 기준 레이블 자동 결정: 9.0 이상=완벽 / 8.0 이상=매우 좋음 / 7.0 이상=좋음 / 6.0 이상=괜찮음 / 6.0 미만=보통 / null=평점 배지·레이블 모두 미표시.
  • numberOfReviews는 "검증된 후기 N개" 형식 표시.
  • reviews 데이터 자체가 없으면 리뷰 탭 전체 빈 상태 표시.
평점 레이블 정의 — 정책서 6-8절 v0.12
← guestRating / numberOfReviews (단일 검색 reviews)
2
카테고리별 점수
  • 편의시설·청결도·객실·서비스·위치·식사 6개 카테고리 점수를 가로 게이지 바 + 숫자로 표시.
  • 게이지 바 너비는 점수/10 비율.
  • categoryScores[] 데이터 없으면 카테고리 점수 섹션 미노출.
← reviews.categoryScores[] (단일 검색 전용)
리뷰 데이터 소스 — 정책서 6-2절
3
리뷰어 정보
  • 아바타: reviewerName 첫 글자 대문자로 이니셜 자동 생성(색상 랜덤 또는 고정 팔레트).
  • 이름: reviewerName.
  • 날짜: reviewDate (예: "2026년 4월").
  • 숙박 기간: stayDuration (예: "1박 숙박함").
← reviews[].reviewerName / .reviewDate / .stayDuration
4
개별 리뷰 점수
  • reviews[].rating 값 기준 배지 색상: 8.0 이상=초록(#059669) / 7.0~7.9=주황(#d97706) / 7.0 미만=빨강(기본 보라 배지 대신 위험색).
  • null이면 점수 배지 미표시.
← reviews[].rating
5
[리뷰 더보기]
  • MVP: 최초 10개 표시 후 [리뷰 더보기] 탭 시 10개씩 추가 로딩(Infinite scroll 또는 버튼 클릭 방식).

  • API 재호출 여부는 엔지니어 확인 필요(단일 검색 결과 내 페이지네이션 vs 추가 호출).
