⚙ 공통 정책 — 공급사별 포인트 현황
  • 데이터 소스: KAYAK Reporting API — providers 기준으로 OTA별로 집계 (6조)
  • 집계 기준: 지급완료(Approved) 건 기준. 적립예정(Active)·취소·만료(Cancelled) 건은 별도 표시 (정책서 v0.30 9-4절)
  • 포인트 단위: P (포인트), 1P = 1원. KAYAK 정산 기준 KRW로 통일 (6-4절)
  • MVP 조회전용: 포인트율 수동 조정 기능 없음 (Phase 2 후 논의)

주석 — ADM-CB02

1
기준 월 선택
  • 조회하고 싶은 월을 선택. 형식: 연도-월 (예: 2026-06). 최근 12개월 제공
  • 기본값은 오늘 날짜가 속한 이번 달
  • 지급완료(Approved) 기준으로 집계 → 체크아웃 후 자사 내부 기준을 통과한 건만 포함
2
공급사별 집계 테이블
  • 각 OTA(타사 예약 사이트)별로 예약 건수와 포인트 합계를 한 눈에 볼 수 있음
  • 포인트 합계는 P(포인트) 단위로 표시. 1P = 1원 기준
  • 포인트율 평균: PERCENTAGE 타입 OTA에만 계산. FLAT·NONE 타입은 "—"으로 표시
  • NONE 타입 OTA: 포인트가 없어도 예약 건수는 집계됨
어드민 v0.3 6-4절 — 포인트 단위 P, 1P=1원
3
요약 통계 계산 방식
  • 총 포인트(Approved) 1,200,000P: 해당 월 지급완료 건의 포인트를 모두 더한 값
  • 총 예약 건수(312건): 해당 월 예약 완료 건 전체 — 적립예정(Active) + 지급완료(Approved) + 취소(Cancelled) 합산
  • 참여 OTA 수(7개): 해당 월에 예약이 1건 이상 발생한 OTA의 수
  • 적립예정(Active) 340,000P: 해당 월 발생 건 중 아직 체크아웃 전이거나 확정 안 된 포인트 합계
  • 평균 포인트율 3.8%: PERCENTAGE 타입 OTA에서 (포인트 합계 ÷ 예약금액 합계) × 100. FLAT·NONE 타입 OTA 제외
어드민 v0.3 6-4절
4
포인트율/금액 컬럼 — 타입별 표시 방식
  • PERCENTAGE 타입: 포인트율(%)만 표시 — 예: 3.2% (예약금액의 3.2%를 포인트로 줌)
  • FLAT 타입: 고정 포인트 금액만 표시 — 예: 240,000P (예약금액과 무관하게 항상 이 금액을 줌)
  • NONE 타입: "—" 표시 — 포인트 없는 OTA
  • 두 값(% + 금액)을 한 행에 같이 표시하지 않음. 타입에 따라 하나만 표시
어드민 v0.3 6-4절 — KAYAK 포인트 타입 구분
포인트율 조정 공급사별 포인트율 조정은 KAYAK 담당자에게 양식을 제출하는 방식으로 진행.
어드민 화면에서 직접 변경할 수 없음. Phase 2에서 조정 이력 조회 기능 논의 예정.
