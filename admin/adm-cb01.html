⚙ 공통 정책 — 포인트 내역
  • 데이터 소스: KAYAK Reporting API — labels 파라미터로 회원에게 귀속된 포인트 내역을 가져옴 (6조)
  • 표시 대상: TypeCode=10(예약 완료 건)만 표시. TypeCode=1(클릭 건)은 표시하지 않음 (6-2절)
  • MVP 조회전용: 포인트 지급·삭제·회수 기능은 없음. Phase 2에서 추가 예정 (6조)
  • 지급완료(Approved) 판단: 자사 내부 기준 적용 — 체크아웃 후 일정 기간 경과 시 자동 전환. KAYAK paymentMonth 값 단독 기준 아님 (6-2절 v0.42)
  • 적립예정(Active) 주의: KAYAK API 상태값은 Active. 체크아웃 전 미확인 상태 (6-2절)
  • 취소·만료(Cancelled): TransactionStatus = Cancelled. PERCENTAGE 타입이어도 포인트 금액 0인 실데이터 케이스 있음 (6-2절)
  • NONE/0P 건 미표시: cashbackType = NONE 또는 포인트 금액 0인 건은 이 화면에 표시하지 않음. 예약내역(ADM-BOOK01)에서만 확인 가능 (6-2절)
  • 금액 단위: P (포인트). 1P = 1원 기준. 소수점 이하 버림 (예: 7,659.42 → 7,659P) (6-4절 / 포인트_정책서 v0.1 2-2절)
  • KAYAK 정산 상태 표시 방식: TransactionStatus(Active/Cancelled) 어드민 별도 표시 여부 미결 — 개발팀 확인 필요 (8조 미결 9)

Phase 2 예정 기능: 포인트 수동 지급 / 삭제 / 회수 기능은 Phase 2에서 추가. MVP에서는 조회만 가능.

주석 — ADM-CB01

1
검색 필드
  • 회원 이메일 주소 또는 예약 ID(예: RES-20001)로 특정 내역을 찾을 수 있음
  • 회원 상세 화면(ADM-MEM02)의 포인트 탭과 같은 데이터를 보여줌. 이 화면은 전체 회원 대상으로 한 번에 조회하는 점이 다름
2
상태 필터
  • 포인트 상태를 세 가지로 구분해 필터링할 수 있음
  • 적립예정 (Active): 아직 체크아웃 전이거나 확인이 안 된 상태. KAYAK 시스템에서 가져오는 상태값은 Active
  • 지급완료 (Approved): 자사 내부 기준(체크아웃 후 일정 기간 경과)에 따라 확정된 상태. KAYAK paymentMonth 값만으로 판단하지 않음 — 자사 정책 우선 적용
  • 취소·만료 (Cancelled): 예약이 취소됐거나 포인트가 만료된 상태. PERCENTAGE 타입이어도 포인트 금액이 0인 실데이터가 존재하므로 타입만으로 금액을 추측하면 안 됨
  • 포인트 금액이 0이거나 타입이 NONE인 건은 이 화면에 나타나지 않음. 확인하려면 예약내역(ADM-BOOK01) 화면을 이용
어드민 v0.3 6-2절 — Active/Approved/Cancelled 상태 정의 및 Approved 판단 기준 (v0.42)
3
지급월 필터
  • 특정 월의 지급완료 건만 보고 싶을 때 입력. 형식: 연도-월 (예: 2026-06)
  • 지급완료(Approved) 상태인 건에만 적용됨. KAYAK 기준 미국 동부 시간(EST) 기준이므로 한국 시간과 최대 13~14시간 차이가 날 수 있음
4
포인트 타입 배지
  • PERCENTAGE: 예약금액의 몇 % 를 포인트로 줌 (예: 3% → ₩320,000 예약 시 9,600P 적립)
  • FLAT: 예약금액과 상관없이 정해진 포인트를 그대로 줌
  • NONE: 포인트 없음 — 이 화면에 표시하지 않음. 예약내역(ADM-BOOK01)에서만 확인 가능
  • 주의: PERCENTAGE 타입이어도 예약이 취소(Cancelled)되면 포인트 금액이 0이 될 수 있음. 타입 배지만 보고 포인트 금액을 추측하면 안 됨 (2026-07-06 실데이터 확인)
  • 포인트 금액 소수점 처리: KAYAK API에서 소수점 있는 값 내려올 수 있음 (예: 7,659.42). 화면 표시·DB 저장 모두 소수점 이하 버림 (7,659P)
어드민 v0.3 6-2절 / 6-4절 — NONE 미표시 + 실데이터 주의사항
5
요약 통계 계산 방식
  • 누적 포인트(Approved) 12,400,000P: 지급완료 상태인 모든 건의 포인트를 더한 값. 단위는 P(포인트), 1P = 1원
  • 적립예정(Active) 2,100,000P: 아직 체크아웃 전이거나 확인 중인 건의 포인트 합계. 최종 지급이 확정되지 않은 예정 금액
  • 취소·만료(Cancelled) 340,000P: 취소되거나 만료된 건의 포인트 합계. 이 금액은 환불되지 않고 자사 수익으로 처리됨
  • 귀속 회원 수 872명: 포인트가 1건 이상 적립된 로그인 회원 수. 로그인 없이 이용한 경우는 귀속되지 않으므로 이 숫자에 포함되지 않음
어드민 v0.3 6-4절 — 포인트 단위 P, 1P=1원
