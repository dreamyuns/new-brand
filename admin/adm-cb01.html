⚙ 공통 정책 — 캐시백 내역
  • 데이터 소스: KAYAK Reporting API labels 파라미터로 회원 귀속 캐시백 조회 (6조)
  • MVP 조회전용: 지급·삭제·회수 기능 없음. Phase 2에서 추가 (6조)
  • Approved(지급완료) 판단: TransactionStatus = Active AND PaymentMonth 값 존재. 2026-06-23 KAYAK 공식 확인 (6-2절)
  • Cancelled(취소·만료) 주의: TransactionStatus = Cancelled. PERCENTAGE 타입이어도 금액 0인 케이스 실데이터에서 확인됨 (6-2절)
  • NONE/0원 건 미표시: cashbackType = NONE 또는 캐시백 금액 0원인 건은 캐시백 내역에 표시하지 않음. 예약내역(ADM-BOOK01)에서만 조회 가능 (6-2절)
  • 지급월 기준: EST(미국 동부 시간) 기준. 표시 시 한국 시간과 최대 13~14시간 차이 존재 (6-2절)
  • 금액 통화: KAYAK API 응답 통화 그대로 표시. 자사 환산 없음 (9-1절)

주석 — ADM-CB01

1
검색 필드
  • 회원 이메일 또는 예약ID 검색
  • 회원 상세(ADM-MEM02) 캐시백 탭과 동일 데이터, 필터만 다름
2
상태 필터
  • KAYAK Reporting API transactionStatus 기반 3개 값
  • Waiting(적립예정): 체크아웃 전 미확인 상태
  • Approved(지급완료): TransactionStatus = Active + PaymentMonth 값 존재 조건 모두 충족 시. 2026-06-23 KAYAK 공식 확인 완료
  • Cancelled(취소·만료): TransactionStatus = Cancelled. PERCENTAGE 타입이어도 금액 0원인 실데이터 케이스 있음 — 주의
  • NONE/0원 건은 이 화면에 미표시 (예약내역 ADM-BOOK01에서 조회)
어드민 v0.3 6-2절 — Approved 판단 기준 확정
3
지급월 필터
  • YYYY-MM 형식 입력
  • Approved 건의 지급 확정월 기준
4
캐시백 타입 배지
  • PERCENTAGE: 예약금액 대비 % 지급
  • FLAT: 고정 금액 지급
  • NONE: 캐시백 없음 — 이 화면에 표시하지 않음 (예약내역에서만 조회 가능)
  • 주의: PERCENTAGE 타입이어도 Cancelled 상태일 때 캐시백 금액 0원인 실데이터 케이스 확인됨 (2026-07-06 실데이터). 타입만으로 금액 유추 불가
어드민 v0.3 6-2절 / 6-4절 — NONE 미표시 + 실데이터 주의사항
5
요약 통계 계산식
  • 누적 캐시백(Approved) ₩12,400,000: Approved 건 캐시백 금액 전체 합산. 현재 KRW 기준 (다중 통화 혼재 시 통화별 분리 표시 필요 — 개발팀 협의)
  • 적립예정(Waiting) ₩2,100,000: Waiting 건 캐시백 금액 합산 (체크아웃 전 미확인)
  • 취소·만료(Cancelled) ₩340,000: Cancelled 건 합산. 환불 없음, 자사 수익으로 귀속
  • 귀속 회원 수 872: 캐시백 1건 이상 귀속된 로그인 회원 수. 비로그인 미귀속 건은 제외
통화 혼재 시 집계 기준 미결 — 개발팀 협의 필요 / 정책서 v0.30 9절
