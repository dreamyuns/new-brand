제1조 목적 및 MVP 범위
1-1. 서비스 목적

본 정책서는 신규 메타서치 MVP 어드민의 기능·권한·화면 정책을 정의한다.
어드민은 내부 운영팀이 사용하는 관리 도구로, 고객 대면 서비스(프런트)와 독립적으로 운영된다.

지급월PaymentMonthEST(미국 동부 시간) 기준. 한국 시간과 최대 13~14시간 차이
상태 배지내부 정의적립예정 / 지급완료 / 취소·만료

지급완료 판단 기준 (포인트_정책서 v0.1 2-1절 확정):
자사 내부 기준 — 체크아웃일로부터 7일 경과 후 자동으로 지급완료 전환 (자사 배치 처리). KAYAK PaymentMonth 단독 기준이 아님.
TransactionStatus = Active AND 자사 체크아웃+7일 조건 충족 시 지급완료 처리.

Cancelled(취소·만료) 주의사항 (2026-07-06 실데이터 확인):
TransactionStatus = Cancelled. PERCENTAGE 타입이어도 포인트 금액 0원인 실데이터 케이스 존재. 타입만으로 금액 유추 불가.

NONE/0원 건 미표시:
CashbackType = NONE 또는 포인트 금액 0원인 건은 포인트 내역 화면에 표시하지 않음. 예약내역(ADM-BOOK01)에서만 조회 가능.

6-4. 공급사별 포인트 현황

공급사 코드별 포인트 발생 건수·합계 금액 집계. 기간 필터 적용 가능.

⚠️ CPC 예외 공급사: 일부 공급사는 CPA(예약 완료 시 수수료) 대신 CPC(클릭 시 수수료) 모델로 운영됨. CPC 건은 TransactionType=Click으로 분류되어 예약내역에 표시되지 않음. 공급사별 수익 분석 시 이 점 고려 필요.
