⚙ 공통 정책 — 예약내역
  • 데이터 소스: KAYAK Reporting API — TypeCode=10(예약 완료 건)만 표시. TypeCode=1(클릭 건)은 표시하지 않음 (5조)
  • 회원 식별: 아웃링크 URL에 포함된 p= 파라미터(회원 ID)를 Reporting API의 labels 필드와 연결해 회원별로 구분 (정책서 v0.30 9절)
  • 포인트 지급완료(Approved) 판단: 자사 내부 기준 적용 — 체크아웃 후 일정 기간 경과 시 자동으로 지급완료 처리. KAYAK paymentMonth 값 단독 기준 아님 (5조 v0.42)
  • MVP 조회전용: 상태 변경·환불 처리 기능 없음. Phase 2에서 추가 예정 (5조)
  • Excel 다운로드: 개발 공수 확인 후 결정. 미확정 시 Phase 2 이동 (8조 미결 3)

주석 — ADM-BOOK01

1
통합 검색
  • 예약 ID(예: RES-20001), 회원 이메일, 호텔 이름 중 하나로 원하는 내역을 찾을 수 있음
  • KAYAK에서 받아온 데이터를 자사 서버에 저장한 뒤 검색하거나, 직접 API에서 필터링함 (개발팀 결정 사항)
2
상태 필터
  • 예약 건의 포인트 상태를 세 가지로 구분해 필터링할 수 있음
  • 적립예정 (Active): 아직 체크아웃 전이거나 포인트 지급이 확정되지 않은 상태. KAYAK 시스템 상태값은 Active
  • 지급완료 (Approved): 자사 내부 기준(체크아웃 후 일정 기간 경과)에 따라 포인트 지급이 확정된 상태
  • 취소·만료 (Cancelled): 예약이 취소됐거나 포인트가 만료된 상태
어드민 v0.3 5조 / 정책서 v0.30 9-4절 — 포인트 상태값 (v0.42 Approved 기준 변경)
3
공급사(OTA) 필터
  • KAYAK에서 제공한 OTA(타사 예약 사이트) 이름 목록으로 필터링. 예: Agoda, Booking.com, Hotels.com
  • 고정 목록이 아니라 실제 데이터에 있는 OTA 이름을 동적으로 불러와 보여줌
4
Excel 다운로드
  • 지금 화면에 적용된 필터 조건 그대로 전체 데이터를 엑셀 파일로 내려받음
  • 포함 항목: 예약ID · 회원 · 호텔 · 도시 · OTA · 체크인/아웃 · 예약금액 · 포인트 · 상태
개발 공수 확인 후 결정 (8조 미결 3)
5
비로그인 미귀속 건
  • 로그인 없이 타 OTA로 이동한 경우, 예약이 완료돼도 어떤 회원이 예약했는지 알 수 없음
  • 이런 경우 포인트 수익은 자사가 받지만 특정 회원의 포인트로 쌓아줄 수 없음 → 회원 컬럼에 "미귀속" 표시
  • 이를 줄이기 위해 아웃링크 전 로그인 유도 모달을 운영함
정책서 v0.30 9절 — 비로그인 포인트 귀속 불가
6
요약 통계 계산 방식
  • 전체 예약(1,203건): KAYAK Reporting API에서 가져온 예약 완료 건(TypeCode=10)의 전체 개수
  • 적립예정(287건): KAYAK 상태값 Active — 아직 체크아웃 전이거나 포인트 미확정 건
  • 지급완료(891건): 자사 기준으로 포인트 지급이 확정된 건 (Approved)
  • 취소·만료(25건): 취소 또는 만료된 건 (Cancelled)
  • 검산: 1,203 = 287 + 891 + 25 ✓
KAYAK Reporting API transactionStatus 필드 / 어드민 v0.3 5조
7
예약금액(상품가) 컬럼
  • KAYAK Reporting API의 transactionAmount 값 — 고객이 OTA(타사 예약 사이트)에 실제로 결제한 금액
  • 세금·수수료 포함 여부는 OTA마다 다름 — 화면에서는 "상품가"로 통일해 표기
  • 통화가 다를 수 있음: 원화(₩), 엔화(¥), 태국 밧(THB), 싱가포르 달러(SGD) 등 그대로 표시. 포인트는 별도 P 단위로 표기
어드민 v0.3 5-2절 — 예약금액(상품가) 라벨 확정
