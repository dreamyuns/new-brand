진입
마이페이지 > [캐시백 내역] 탭 → 별도 전체화면
  • 로그인 전용 화면. 비로그인 회원은 MY01에서 로그인 유도 후 진입 가능.
  • 진입 시 KAYAK Transaction Report API를 호출해 해당 회원의 전체 캐시백 내역을 로드.
  • ⚠️ 방식 2(고객 직접 지급) Phase 2 — MVP에서는 KAYAK Transaction Report 기반 내역 추적만 제공. 실제 캐시백 지급은 Phase 2 도입 후 활성화. (정책서 9-1절·9-6절 v0.21)
정책서 10-2절. 로그인 상태 전용. ⚠️ 방식 2 Phase 2.

2
총 적립 포인트 패널 ★ (상단 고정)
  • 지급완료 상태 건(자사 체크아웃+7일 배치 처리 완료)의 포인트만 누적 합산해 표시.
  • 적립예정·취소·만료 건은 합산에서 제외.
  • 내역이 없거나 지급완료 건이 없으면 0P 표시 (패널 자체는 항상 노출).
  • 금액 단위: P (포인트). 소수점 이하 버림 값으로 표시.
← 자사 포인트 DB에서 event_type=booking_earn 처리 완료 건의 point_delta 합산 (cashbackAmountLocalised 소수점 버림 적용 후 값).
포인트_정책서 v0.1 5-1절 / 정책서 10-2절 (상단 고정 총 적립 포인트 — 지급완료 기준).
3
전체 내역 건수
  • 적립예정·지급완료·취소·만료 모든 상태 건수를 합산해 표시 (예: "전체 내역 3건").
  • 정렬 기준: 지급완료일(자사 배치 처리 완료일) 최신순. 적립예정 건(미지급)은 목록 하단 배치.
정책서 10-2절 (지급완료일 최신순, 페이지네이션 없음).
4
호텔명
  • KAYAK Transaction Report의 호텔명 필드 표시. 긴 이름은 말줄임(ellipsis) 처리.
  • 카드 헤더 좌측 고정, 상태 배지와 같은 행에 배치.
← hotelName (Transaction Report Hotel vertical 필드)
5
상태 배지 — 지급완료
  • 자사 체크아웃+7일 배치 처리가 완료된 건. 초록 배지.
  • 배지 표기: "지급완료" (초록 배경 #d1fae5, 텍스트 #065f46).
← 자사 포인트 DB event_type=booking_earn 처리 완료 건 (KAYAK Reporting API 기반 체크아웃+7일 배치).
배지 3종: 적립예정(노랑)·지급완료(초록)·취소·만료(회색). 포인트_정책서 v0.1 2-4절 / 정책서 10-2절.
6
적립 포인트 금액 ★
  • KAYAK Reporting API cashbackAmountLocalised 값을 그대로 사용. 자사에서 재계산하지 않음.
  • 소수점 이하 버림 처리 후 표시 (예: 7,659.42 → 7,659P). DB 저장 시도 버림 적용.
  • 지급완료·적립예정 건: 인디고(#4f46e5) 색상 강조 + "+" 접두사 표시 (예: +7,659P).
  • 취소·만료 건: 금액에 취소선(line-through) + 회색 처리.
← KAYAK Reporting API cashbackAmountLocalised 필드 직접 사용. 실 데이터(2026-07-06) 확인: 소수점 값 존재(예: 7659.42) + Cancelled 건에서 금액 0으로 내려오는 경우 있음 — 취소선 표시 우선.
소수점 이하 버림: 포인트_정책서 v0.1 2-2절. TransactionStatus=Cancelled 기준 취소선 우선. cashbackType=NONE 미표시 정책과 별개. 정책서 9-4절·10-2절.
7
카드 — 적립예정 (Active) 샘플
  • 체크아웃+7일 미도래인 건 (KAYAK TransactionStatus = Active). 노란 배지.
  • 카드 footer 우측 레이블: "예상 포인트" (지급완료 건의 "적립 포인트"와 구분).
← KAYAK Reporting API TransactionStatus = Active (체크아웃+7일 미도래 상태)
8
상태 배지 — 적립예정
  • 체크아웃+7일 미도래 상태. 자사 배치 처리 대기 중인 건. 노란 배지.
  • 배지 표기: "적립예정" (노란 배경 #fef9c3, 텍스트 #854d0e).
포인트_정책서 v0.1 2-4절 (KAYAK Active + 체크아웃+7일 미도래 = 적립예정 배지).
9
카드 — 취소·만료 (Cancelled) 샘플
  • 예약 취소, 미투숙, 또는 만료로 캐시백이 지급되지 않은 건. 회색 배지.
  • 카드 detail 영역에 예약일과 취소일 표시. 예약금액·캐시백 금액은 회색 취소선 처리.
  • ⚠️ 실 데이터 확인(2026-07-06): Cancelled 건에서 캐시백 금액이 0원으로 내려오는 경우 있음. 금액이 0원이어도 취소선 표시를 먼저 적용함 — cashbackType=NONE 미표시 정책과 다른 처리. 예약이 취소됐는지 아닌지를 먼저 보고, 취소됐다면 금액과 관계없이 취소선으로 표시.
← KAYAK Transaction Report 상태값 = Cancelled
10
상태 배지 — 취소·만료
  • 적립 금액에 취소선(line-through) + 회색(#9ca3af) 처리. 총 적립금액 합산에서 제외.
  • 배지 표기: "취소·만료" (회색 배경 #f1f5f9, 텍스트 #64748b).
정책서 9-4절 (Cancelled → 취소선·회색 처리, 합산 제외).

캐시백 카드 전체 API 필드 매핑
  • 아래 모든 필드는 KAYAK Reporting API Transaction Report에서 제공 (정책서 10-2절).

예약일 → bookingDate

적립 포인트 금액 → cashbackAmountLocalised (소수점 이하 버림 적용. null·0·cashbackType=NONE 건 미표시. Cancelled 건은 0이어도 취소선 표시 우선)
